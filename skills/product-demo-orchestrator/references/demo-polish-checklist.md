# Demo Polish Checklist

## Baseline UI

- 标题、描述、CTA 的层级清楚
- 核心模块数量可控
- 页面视觉不漂移

## Accessibility

- 交互元素具备清晰名称
- 焦点状态或可触达提示存在
- 颜色对比度足够

## Metadata

- 页面标题或 demo 名称明确
- 目录与文件命名能让人一眼识别用途
- handoff 文档说明怎么展示

## Motion Performance

- 动效不抢内容注意力
- 默认时长控制在 150-300ms
- 有减少动效策略

## Demo Readiness

- walkthrough 能单独被拿来讲 demo
- 关键页面可以录屏
- 当前限制明确，不把占位实现说成已完成
