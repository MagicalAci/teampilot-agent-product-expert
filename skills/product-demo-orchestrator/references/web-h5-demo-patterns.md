# Web And H5 Demo Patterns

## Web Demo

适合：

- 产品首页 demo
- 落地页
- 数据看板演示
- 销售演示页

默认建议：

- 首屏先讲清价值，再补模块
- 保留 1 个主 CTA，最多 1 个次 CTA
- 优先做可录屏、可截图、可讲述的页面结构

## H5 Demo

适合：

- 移动端主路径演示
- 活动页 / 营销页
- 小程序或 App 功能的近似预演

默认建议：

- 移动端首屏优先级高于“模块数量”
- 目标尺寸先按窄屏优化，再补大屏兼容
- 不要把桌面端信息密度原样塞进 H5

## 共通规则

- 动效优先使用 `transform` 和 `opacity`
- 首版默认用静态数据，不要因为等接口阻塞 demo
- 演示页应与 walkthrough 一起维护，不要两套口径
