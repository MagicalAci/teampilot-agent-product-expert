# Demo Packaging

一个可交付的 demo 包至少应包含：

- brief
- design system
- demo 入口文件
- preflight checklist
- walkthrough
- developer handoff

## 打包前检查

1. 目标平台正确
2. walkthrough 与页面一致
3. design system 与实现一致
4. 已知限制已写清

## 推荐交接顺序

1. 先看 `brief/demo-brief.md`
2. 再看 `design-system/MASTER.md`
3. 再打开 `demo/`
4. 最后看 `review/` 与 `handoff/`
