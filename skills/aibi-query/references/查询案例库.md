## Case 1：用户大盘

**PM 提问：** "目前绘本产品有多少注册用户？"

```sql
SELECT COUNT(*) AS total_users
FROM t_pic_book_user_info
WHERE is_delete = 0
```

**返回字段：** `total_users`

**场景价值：** 每日晨会快速看用户大盘，不用找开发拉数。

## Case 2：设备绑定漏斗

**PM 提问：** "有多少台设备已经绑定用户？绑定和解绑各多少？"

```sql
SELECT
  COUNT(*) AS total_bindings,
  SUM(CASE WHEN status = 1 THEN 1 ELSE 0 END) AS active_bindings,
  SUM(CASE WHEN status = 0 THEN 1 ELSE 0 END) AS unbound
FROM t_pic_book_device_user_binding
WHERE is_delete = 0
```

**返回字段：** `total_bindings`, `active_bindings`, `unbound`

**场景价值：** 追踪激活率。设备卖出去了但没绑定 = 用户流失风险。

## Case 3：每日阅读趋势

**PM 提问：** "最近 7 天每天有多少人在阅读？每天总阅读时长多少分钟？"

```sql
SELECT
  stat_date,
  COUNT(DISTINCT user_id) AS active_readers,
  SUM(total_minutes) AS total_reading_minutes,
  SUM(book_count) AS total_books_read
FROM t_pic_book_reading_daily_stat
WHERE is_delete = 0
  AND stat_date >= DATE_SUB(CURDATE(), INTERVAL 7 DAY)
GROUP BY stat_date
ORDER BY stat_date DESC
```

**返回字段：** `stat_date`, `active_readers`, `total_reading_minutes`, `total_books_read`

**场景价值：** 核心 DAU + 阅读深度指标，观察功能迭代后的数据变化。

## Case 4：热门绘本 TOP 10

**PM 提问：** "用户书架上最多的绘本 TOP10 是哪些？来源是扫码还是上传？"

```sql
SELECT
  book_id,
  book_name,
  COUNT(DISTINCT user_id) AS user_count,
  SUM(CASE WHEN book_source = 0 THEN 1 ELSE 0 END) AS from_isbn_scan,
  SUM(CASE WHEN book_source = 1 THEN 1 ELSE 0 END) AS from_upload
FROM t_pic_book_user_shelf
WHERE is_delete = 0
GROUP BY book_id, book_name
ORDER BY user_count DESC
LIMIT 10
```

**返回字段：** `book_id`, `book_name`, `user_count`, `from_isbn_scan`, `from_upload`

**场景价值：** 内容运营选品依据，了解 ISBN 扫码 vs 用户上传的比例。

## Case 5：AI 对话功能使用情况

**PM 提问：** "AI 对话功能最近 30 天的使用情况怎么样？每天多少人用、多少轮对话？"

```sql
SELECT
  DATE(create_time) AS chat_date,
  COUNT(DISTINCT user_id) AS chat_users,
  COUNT(DISTINCT CONCAT(user_id, '_', round_id)) AS total_rounds,
  COUNT(*) AS total_messages
FROM t_pic_book_conversation_record
WHERE is_delete = 0
  AND create_time >= DATE_SUB(CURDATE(), INTERVAL 30 DAY)
GROUP BY DATE(create_time)
ORDER BY chat_date DESC
```

**返回字段：** `chat_date`, `chat_users`, `total_rounds`, `total_messages`

**场景价值：** AI 功能 engagement 核心指标，判断 AI 对话是否被孩子真正使用。

## Case 6：用户反馈分析

**PM 提问：** "最近的用户反馈按类型和处理状态怎么分布？"

```sql
SELECT
  CASE feedback_type
    WHEN 0 THEN '功能反馈'
    WHEN 1 THEN '内容反馈'
    WHEN 2 THEN '音频反馈'
    WHEN 3 THEN '其他'
  END AS feedback_category,
  CASE status
    WHEN 0 THEN '未处理'
    WHEN 1 THEN '处理中'
    WHEN 2 THEN '已处理'
    WHEN 3 THEN '已关闭'
  END AS handle_status,
  COUNT(*) AS cnt
FROM t_pic_book_feedback
WHERE is_delete = 0
GROUP BY feedback_type, status
ORDER BY feedback_type, status
```

**返回字段：** `feedback_category`, `handle_status`, `cnt`

**场景价值：** 快速定位待处理反馈堆积量，判断是内容问题还是功能 bug。

## Case 7：声纹功能渗透率

**PM 提问：** "有多少用户录制了自己的声纹？生成成功率多少？"

```sql
SELECT
  voiceprint_type,
  COUNT(*) AS total,
  SUM(CASE WHEN generate_status = 1 THEN 1 ELSE 0 END) AS success,
  SUM(CASE WHEN generate_status = 0 THEN 1 ELSE 0 END) AS processing,
  SUM(CASE WHEN generate_status = 2 THEN 1 ELSE 0 END) AS failed,
  COUNT(DISTINCT user_id) AS unique_users
FROM t_pic_book_voiceprint
WHERE is_delete = 0
GROUP BY voiceprint_type
```

**返回字段：** `voiceprint_type`(PARENT/SYSTEM), `total`, `success`, `processing`, `failed`, `unique_users`

**场景价值：** 声纹是差异化功能，渗透率和成功率直接影响留存。

## 扩展：星卡常用查询

**"站姐和普通用户分别多少？"**
```sql
SELECT
  SUM(CASE WHEN permission=1 THEN 1 ELSE 0 END) AS station_masters,
  SUM(CASE WHEN permission=0 THEN 1 ELSE 0 END) AS normal_users,
  COUNT(*) AS total
FROM t_starcard_user WHERE is_delete=0
```

**"活动状态分布？"**
```sql
SELECT
  CASE activity_status WHEN 0 THEN '待审核' WHEN 1 THEN '已发布' WHEN 2 THEN '未通过' WHEN 3 THEN '已结束' WHEN 4 THEN '已关闭' END AS status,
  COUNT(*) AS cnt
FROM t_starcard_activity WHERE is_delete=0 GROUP BY activity_status
```

**"总 GMV 和客单价？"**
```sql
SELECT
  COUNT(*) AS order_count,
  SUM(pay_amount) AS total_gmv,
  AVG(pay_amount) AS avg_price,
  COUNT(DISTINCT user_id) AS unique_buyers
FROM t_starcard_order WHERE is_delete=0
```

**"AI 鉴定通过率？"**
```sql
SELECT
  COUNT(*) AS total_tasks,
  SUM(CASE WHEN ai_result=1 THEN 1 ELSE 0 END) AS passed,
  SUM(CASE WHEN ai_result=0 THEN 1 ELSE 0 END) AS failed
FROM t_starcard_identify_task WHERE is_delete=0
```

**"聊天活跃度（最近7天）？"**
```sql
SELECT
  DATE(create_time) AS chat_date,
  COUNT(DISTINCT user_id) AS active_users,
  COUNT(*) AS message_count,
  SUM(CASE WHEN sender_type=1 THEN 1 ELSE 0 END) AS ai_messages,
  SUM(CASE WHEN sender_type=2 THEN 1 ELSE 0 END) AS user_messages

WHERE is_delete=0 AND create_time >= DATE_SUB(CURDATE(), INTERVAL 7 DAY)
GROUP BY DATE(create_time) ORDER BY chat_date DESC
```

**"订阅收入分布？"**
```sql

  CASE subscription_type WHEN 1 THEN '月度' WHEN 2 THEN '年度' END AS type,
```
