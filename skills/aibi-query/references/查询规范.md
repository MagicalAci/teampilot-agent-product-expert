# DBOPS DQL 查询规范

> 本文档定义了通过 DBOPS API 执行 SQL 查询的完整规范。

---

## 一、API 基础信息

**Base URL：** `https://hdbs-service-api.hellobike.cn`

**认证方式：** 在 Header 中携带 SSO Token
```
Token: bearer_xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx
Content-Type: application/json
```

---

## 二、DQL 查询 API

### 接口

```
POST /api/v1/db/dql/sqlQuery
```

### 请求体（所有参数在 Body 中，非 URL 参数）

```json
{
  "env": "UAT",
  "dbName": "sparklab_starcard",
  "logicDbName": "sparklab_starcard",
  "dbType": "MySQL",
  "sqlContent": "SELECT COUNT(*) AS total FROM t_starcard_user WHERE is_delete=0",
  "limit": 100,
  "queryType": "PHYSICAL",
  "unit": "HZUNIT"
}
```

### 参数说明

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| env | string | ✅ | 环境：`UAT` / `PROD` / `FAT` / `PT` |
| dbName | string | ✅ | 物理库名（通常和 logicDbName 相同） |
| logicDbName | string | ✅ | 逻辑库名 |
| dbType | string | ✅ | 固定 `MySQL` |
| sqlContent | string | ✅ | SQL 语句（**仅允许 SELECT**） |
| limit | int | ✅ | 返回行数上限：100 / 500 / 1000 / 5000 |
| queryType | string | ✅ | 固定 `PHYSICAL` |
| unit | string | ✅ | 数据中心：固定 `HZUNIT` |

### 响应格式

```json
{
  "code": 200,
  "message": "成功",
  "data": {
    "sqlContent": "select ... limit 100",
    "columns": ["col1", "col2"],
    "rows": [
      {"col1": "value1", "col2": "value2"}
    ],
    "rowSize": 1,
    "executeTime": 0.001,
    "instanceList": [
      {
        "dbName": "sparklab_starcard",
        "dbHost": "10.69.26.47",
        "dbPort": 5235,
        "dbType": "MySQL",
        "role": "slave",
        "dcName": "HZUNIT"
      }
    ]
  },
  "ok": true
}
```

## 三、硬性限制

| 规则 | 说明 |
|------|------|
| **仅 SELECT** | 只允许 SELECT 语句。SHOW / INSERT / UPDATE / DELETE / DDL 均被拦截，返回 "当前语句非select语句" |
| **自动加 LIMIT** | 系统在 SQL 末尾自动追加 `LIMIT N`（N 取决于请求中的 limit 字段） |
| **只读从库** | 查询走 slave 节点（role=slave），不影响主库写入 |
| **SSO 认证** | 每次请求必须携带有效 SSO Token，过期返回 401 |
| **超时** | 建议 curl 加 `--max-time 30`，复杂查询可加到 60 |

---

## 四、错误码处理

| code | message | 含义 | 处理方式 |
|------|---------|------|----------|
| 200 | 成功 | 查询成功 | 解析 data.rows |
| 401 | 用户未登录或登录信息已过期 | Token 过期 | 引导用户重新获取 Token |
| 400 | 当前语句非select语句 | SQL 不合规 | 改为 SELECT 语句 |
| 400 | sql不能为空 | sqlContent 为空 | 检查请求体 |
| 400 | 查询数据库连接信息异常 | 该库在当前 env 无物理实例 | 确认库名拼写正确，联系 DBA 检查实例状态 |
| 400 | null | 请求体格式错误 | 检查 JSON 格式 |

## 五、辅助 API

| API | 方法 | 用途 | 参数 |
|-----|------|------|------|
| `/api/v1/db/logicDatabases` | GET | 查询逻辑库列表 | env, logicDbName |
| `/api/v1/db/physicalSources` | GET | 查询物理源信息 | env, dbName |
| `/api/v1/db/databases/tableList` | GET | 查询表列表 | queryType, dbType |
| `/api/v1/db/databases/tableFieldList` | GET | 查询表字段结构 | — |
| `/api/v1/db/dql/sqlExplainQuery` | POST | SQL 执行计划 | 同 sqlQuery |
| `/api/v1/db/dql/getSqlRelateTableStructureInfo` | POST | 获取 SQL 关联表结构 | — |
| `/api/v1/db/dql/querySqlHistory` | GET | 查询 SQL 历史 | — |
| `/api/v1/db/dql/collectSqlRecord` | GET | 收藏的 SQL 记录 | — |

所有辅助 API 同样需要 `Token` Header。

## 六、查询最佳实践

### 6.1 先查元数据再写 SQL

```sql
SELECT TABLE_NAME, COLUMN_NAME, COLUMN_TYPE, COLUMN_COMMENT
FROM information_schema.COLUMNS
WHERE TABLE_SCHEMA = '库名' AND TABLE_NAME = '表名'
ORDER BY ORDINAL_POSITION
```

### 6.2 显式指定字段

```sql
-- ✅ 正确
SELECT guid, user_new_id, nickname, permission FROM t_starcard_user WHERE is_delete=0

-- ❌ 避免
SELECT * FROM t_starcard_user
```

### 6.3 合理设置 LIMIT

| 场景 | limit 值 |
|------|----------|
| 探索性查询 / 验证 SQL | 100 |
| 统计聚合（COUNT/SUM/AVG） | 100 |
| 明细数据导出 | 5000 |
| 趋势数据（按天） | 1000 |

### 6.4 软删除过滤

**所有查询必须加 `WHERE is_delete = 0`**，否则会包含已删除数据。

### 6.5 时间范围筛选

```sql
-- 最近 N 天
WHERE create_time >= DATE_SUB(CURDATE(), INTERVAL 7 DAY)

-- 指定日期范围
WHERE create_time BETWEEN '2026-03-01' AND '2026-03-31'

-- 按天聚合
GROUP BY DATE(create_time)
```

### 6.6 数据密级注意

- 字段 COLUMN_COMMENT 中标注了 L1-L4 / S2 密级
- L1-L2：普通数据，可直接输出
- L3+：敏感数据（手机号、身份证等），输出时需脱敏
- S2+：高敏感数据，慎重处理

### 6.7 环境选择

| env | 用途 | 数据量 |
|-----|------|--------|
| **UAT** | **线上环境（默认使用）** | **完整真实业务数据** |
| PROD | 备用环境 | 视部署情况 |
| FAT | 功能测试环境 | 极少 |

**重要：** UAT 即我们的线上数据环境，所有查询默认使用 `env=UAT`。

## 七、curl 命令模板

### 基本查询

```bash
curl -s --max-time 30 \
  'https://hdbs-service-api.hellobike.cn/api/v1/db/dql/sqlQuery' \
  -X POST \
  -H 'Content-Type: application/json' \
  -H "Token: $(cat $HOME/.aibi/token)" \
  -d '{
    "env": "UAT",
    "dbName": "sparklab_starcard",
    "logicDbName": "sparklab_starcard",
    "dbType": "MySQL",
    "sqlContent": "你的SQL",
    "limit": 100,
    "queryType": "PHYSICAL",
    "unit": "HZUNIT"
  }'
```

### 查询并格式化输出

```bash
curl -s --max-time 30 \
  'https://hdbs-service-api.hellobike.cn/api/v1/db/dql/sqlQuery' \
  -X POST \
  -H 'Content-Type: application/json' \
  -H "Token: $(cat $HOME/.aibi/token)" \
  -d '{...}' \
  | python3 -m json.tool
```

### 验证 Token

```bash
curl -s --max-time 10 \
  'https://hdbs-service-api.hellobike.cn/api/v1/db/logicDatabases?env=UAT&logicDbName=sparklab_starcard' \
  -H "Token: $(cat $HOME/.aibi/token)" \
  | python3 -c "import sys,json; d=json.load(sys.stdin); print('VALID' if d.get('code')==200 else 'EXPIRED')"
```
