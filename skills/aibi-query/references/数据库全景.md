**t_starcard_user** — 用户表

**t_starcard_activity** — 活动表
- `guid` (PK), `user_id` (发起人), `activity_title`, `product_amount`, `stock_quantity`, `purchase_quantity`, `activity_status` (0待审核/1已发布/2未通过/3已结束/4已关闭)

**t_starcard_identify_task** — AI 鉴定任务
- `guid` (PK), `user_id`, `template_id`, `ai_result` (0不通过/1通过), `ai_reason`, `ai_front_result`, `ai_back_result`, `ai_frontTexture_result`, `ai_backLogo_result`, `ai_pixel_result`

**t_starcard_star_info** — 明星信息
- `guid` (PK), `star_name`, `category` (0内娱/1韩娱/2泰娱/3欧美)

**t_starcard_user_star_relation** — 用户-明星关系
- `user_id`, `star_id`, `is_joined`, `is_followed`

## 五、通用表结构规范

所有表遵循以下公约：

| 字段 | 类型 | 说明 |
|------|------|------|
| `guid` 或 `id` | bigint unsigned | 主键（雪花算法） |
| `create_time` | timestamp | 创建时间 |
| `update_time` | timestamp | 更新时间 |
| `is_delete` | smallint | 软删除标记（0=正常, 1=删除） |

**数据密级标注：** 字段 COLUMN_COMMENT 中包含 L1-L4 / S2 密级标记，L3+ 和 S2+ 为敏感数据，输出时需脱敏。

## 六、如何查看任意表的完整字段

```sql
SELECT COLUMN_NAME, COLUMN_TYPE, COLUMN_COMMENT
FROM information_schema.COLUMNS
WHERE TABLE_SCHEMA = '库名' AND TABLE_NAME = '表名'
ORDER BY ORDINAL_POSITION
```
