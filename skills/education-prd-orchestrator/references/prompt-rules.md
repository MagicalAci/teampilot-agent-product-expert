# Prompt 规则

## 适用范围

本文件适用于这个 skill 下的所有内部 agent。

## 统一写作口径

- 先结论，后展开
- 先证据，后判断
- 不写工具过程
- 不写 AI 自己的思考过程
- 不混用术语

## 正文规则

- PRD 正文必须是正式交付文风，不写“我猜”“可能”“先这样”
- 如果某个判断证据不足，要降级成“待确认”，不要伪装成结论
- 如果一个概念已经冻结，全文只用一个叫法

## 章节输出结构

每个 agent 输出都尽量遵守这 5 段：

1. 本轮结论
2. 主要证据
3. 产出内容
4. 风险与待确认
5. 建议下一步

## 配图规则

- 图只服务于正文的稳定结论
- 一张图只回答一个主问题
- 先明确图要解决什么理解问题，再开始画
- 流程图回答“从哪里到哪里”
- 架构图回答“有哪些模块，关系是什么”

## 用户回问规则

回问不要泛泛而谈，要具体。

要明确：

- 缺哪一类材料
- 需要用户做什么动作
- 为什么这一步不能跳过
- 补完后会继续哪一阶段

## 禁止项

- 禁止发明不存在的当前能力
- 禁止假设未看过的截图内容
- 禁止把用户没体验过的链路写成事实
- 禁止让图和正文采用两套定义
- 禁止跳过版本同步与索引更新
