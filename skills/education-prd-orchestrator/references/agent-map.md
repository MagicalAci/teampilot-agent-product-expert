# Agent Map

## Evidence Agent

### 目标

先把“能证明什么、不能证明什么”整理清楚。

### 输入

- 当前 PRD
- 会议纪要
- 截图
- CSV
- XLSX
- 话术或知识库文档
- 搜索记录或证据日志

### 动作

1. 盘点证据源
2. 必要时运行统计脚本
3. 输出证据摘要和缺口清单
4. 判断是否能进入定义闸门

### 输出

- 证据包摘要
- 缺口清单
- 建议下一步

## Definition Agent

### 目标

在正文写作前冻结产品定义。

### 必须回答

- 一句话定义
- 各端角色
- 第一版主链路
- 什么不是这个产品

### 产出建议

- 一段总定义
- 一张判断表
- 1-3 条排除项

### 闸门规则

只要定义还有分叉，就不进入第四章和第五章。

## Chapter 1 Agent

### 目标

写“当前产品分析”。

### 重点

- 当前端侧现状
- 当前能力长什么样
- 现阶段判断
- 可引用截图

### 完成标准

- 当前现状说清楚
- 不是空泛描述
- 有真实材料支撑

## Chapter 2 Agent

### 目标

写“需求场景还原与能力对位”。

### 重点

- 家长 / 孩子真实问题
- 当前产品能接住什么
- 主要缺口是什么
- 当前信息缺口是什么

### 完成标准

- 场景不是凭空假设
- 每个场景都有现有能力对位

## Chapter 3 Agent

### 目标

写“产品定义与双端关系”。

### 重点

- 产品到底是什么
- 双端关系是什么
- 为什么不是聊天同步
- 深服务在什么位置

### 完成标准

- 这一章能作为后面章节唯一上游口径

## Chapter 4 Agent

### 目标

写“V1 范围与边界”。

### 重点

- 做什么
- 不做什么
- 条件性纳入什么
- 完成标准是什么

### 完成标准

- 可以直接指导取舍
- 不是愿望清单

## Chapter 5 Agent

### 目标

写“核心功能定义”。

### 重点

- 模块拆分
- 服务对象
- 解决问题
- 页面落点
- 核心输出

### 完成标准

- 功能之间的关系清楚
- 能支撑配图和后续实现

## Diagram Agent

### 目标

生成总览图和章节配图。

### 默认类型

- 流程总览图
- 功能架构图
- 必要的章节结构图

### 规则

- 先有稳定结论，再画图
- 一张图只回答一个核心问题
- SVG 为源文件，PNG 为文档引用文件
- 默认输出到独立交付目录的 `images/`，如有仓库内镜像路径，再额外同步

## Sync Agent

### 目标

收口版本、日志、索引和引用。

### 动作

1. 更新主文档版本号和变更记录
2. 更新独立交付目录下的 review / handoff 说明
3. 检查图片引用
4. 如果宿主工作区存在产品索引，再额外同步 `docs/workspaces/product/INDEX.md`
5. 输出完成说明

### 完成标准

- 文档、图片、review 交付说明三者一致
- 如果存在仓库内镜像路径，镜像内容与独立交付目录一致
