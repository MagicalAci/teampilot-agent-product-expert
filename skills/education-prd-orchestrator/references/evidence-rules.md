# 证据规则

## 核心原则

- 先证据，后判断
- 一条关键判断至少能回指到一个真实来源
- 证据强度不够时，必须降级成 `待确认` 或 `线索`
- 搜索结果页本身通常不是最终证据，落地页、原评论、原截图才更接近证据

## 证据强度分级

| 等级 | 来源 | 允许支持的结论 |
|---|---|---|
| `strong` | 官方页面、真实截图、原始结构化数据、一手体验记录 | 可以支持主判断 |
| `medium` | 用户原话、平台评论、经过核对的二手摘要 | 可以支持子判断，但最好配合其他证据 |
| `weak` | 聚合页、转述、样本不足的搜索片段 | 只能做线索或待确认 |

## 证据入稿规则

- 文中每个强判断都要知道自己来自哪个证据包条目
- 如果证据需要截图或文件支撑，`evidence/` 目录里必须能找到
- 如果证据只存在于对话里，没有文件化，就要先落到 `evidence-log.csv` 或 `review/` 说明里
- 不能把“可能、也许、看起来”写成确定事实

## 搜索结果降级规则

- 只有搜索摘要，没有打开落地页：`待确认`
- 只有二手媒体汇总，没有原贴：`待确认`
- 只有少量样本，想下趋势结论：降为“现象观察”
- 只有一张图，没有时间、来源、链接：不能算正式图证

## reviewer 怎么看

- 看判断是否能回到 `evidence/` 中的明确条目
- 看图证是否有来源、截图或原链接
- 看是否有人为补写“待确认”，而不是把弱证据伪装成硬证据
