# 用户旅程图方法论

## 什么是用户旅程图

用户旅程图是对用户从首次接触到持续使用（或离开）的完整体验路径的可视化描述。它不只是功能流程图，而是包含用户在每个阶段的行为、想法、情绪和接触点。

## 何时使用

- 用户画像完成后，理解画像在真实场景中的完整体验
- 识别体验断点、情绪低谷和关键转化节点
- 发现现有产品的缺口和优化机会
- 作为核心痛点抽象和方案构思的输入

## 旅程图构建步骤

### Step 1：选定画像和场景

- 选择一个主画像
- 明确旅程的起点和终点
- 确定旅程的时间跨度（一次使用 / 一天 / 首月）

### Step 2：划分阶段

典型产品旅程阶段：

| 阶段 | 说明 |
|------|------|
| 认知 | 用户如何发现产品的存在 |
| 考虑 | 用户评估是否使用、与替代方案比较 |
| 首次体验 | 注册、onboarding、第一次核心操作 |
| 常规使用 | 日常使用的典型路径 |
| 深度使用 | 解锁高级功能、个性化设置 |
| 付费/转化 | 付费决策或其他关键转化 |
| 留存/流失 | 持续使用或放弃的分岔 |

不一定每个产品都要覆盖全部阶段，选择和当前策划目标相关的阶段。

### Step 3：填充每个阶段

每个阶段记录 5 个维度：

| 维度 | 描述 |
|------|------|
| **用户行为** | 用户在这个阶段做了什么 |
| **接触点** | 通过什么渠道或界面与产品交互 |
| **用户想法** | 用户内心在想什么、在评估什么 |
| **情绪曲线** | 满意/中性/沮丧/困惑 |
| **痛点与机会** | 哪里卡住了、哪里有提升空间 |

### Step 4：标注关键时刻

关键时刻（Moments of Truth）：

- **Aha Moment**：用户第一次感受到产品核心价值的瞬间
- **决策时刻**：用户决定继续/放弃/付费的节点
- **摩擦点**：用户卡住、困惑或放弃的位置
- **情绪峰值**：用户最满意或最沮丧的时刻

### Step 5：识别缺口与机会

对比"用户期望的体验"和"当前实际体验"：

- 哪些阶段缺少产品覆盖
- 哪些接触点体验差
- 哪些阶段用户流失最严重
- 哪些关键时刻没有被设计

## 旅程图输出格式

推荐使用表格 + 情绪曲线的方式呈现：

```
阶段    | 认知       | 首次体验      | 常规使用      | 付费转化
--------|-----------|-------------|-------------|----------
行为    | 搜索、推荐   | 注册、引导     | 核心功能使用   | 对比方案、决策
接触点   | 应用商店    | App 首页      | 核心页面      | 付费页
想法    | "这能解决吗" | "能不能快点"   | "确实有用"    | "值不值"
情绪    | 🤔 期待    | 😐 观望      | 😊 满意      | 🤔 犹豫
痛点    | 不确定价值   | 引导太长      | —           | 价格不透明
机会    | 价值前置    | 缩短到3步     | —           | 免费试用
```

## 多角色旅程

如果产品涉及多角色（如教师+学生、商家+消费者），需要：

- 每个角色画一条旅程线
- 标注角色之间的交互节点
- 识别角色间的体验断裂

## 常见错误

- 画成功能流程图，缺少情绪和想法维度
- 只画 happy path，忽略异常和流失路径
- 没有真实数据支撑，全靠想象
- 旅程阶段划分过于笼统
- 画完就归档，没有用来指导后续设计决策

## 产出物

- 用户旅程图（表格或可视化）
- 关键时刻标注
- 痛点与机会列表（按阶段归组）
- 如涉及多角色：角色交互节点标注
