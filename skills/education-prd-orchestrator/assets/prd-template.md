## 第五章 功能定义

> **核心判断**
>
> [先给本章一句总判断：核心功能模块是什么、关系是什么]

### 5.1 功能架构

功能模块划分、模块间关系和数据流向。

### 5.2 功能需求详情

按模块拆分为独立表格，每个模块内按功能点逐行展开。

**固定表格格式**：

```
| 模块 | 需求描述 | 原型图 |
|------|---------|--------|
```

**需求描述列的写法规范**：

使用 `<br>` 换行 + 全角空格 `　` 缩进的嵌套编号结构，按以下层级逐层拆解：

```
**1. 入口**<br>
　说明从哪里进入本功能<br><br>
**2. 页面组成（从上到下）**<br>
　2.1 组件 A（描述内容、状态、数据来源）<br>
　2.2 组件 B<br><br>
**3.【交互元素名称】**<br>
　3.1 样式要求（仅限功能性描述，不写色值/字号/圆角等设计规格）<br>
　3.2 点击后 → 跳转/弹出/调起什么<br>
　3.3 结果分支<br>
　　3.3.1 成功 → 下一步处理<br>
　　　a. 条件 A → 行为 A<br>
　　　b. 条件 B → 行为 B<br>
　　3.3.2 失败 → 错误提示文案 + 重试逻辑<br>
　　3.3.3 取消 → 返回行为<br><br>
**4. 边界与异常**<br>
　4.1 空状态展示<br>
　4.2 离线处理<br>
　4.3 内容审核策略
```

**每个功能点必须覆盖**：

1. 入口：从哪里进入、触发条件
2. 页面组成：从上到下列出所有可见元素，说明内容和数据来源
3. 交互规则：每个可操作元素的点击/输入行为、条件分支（成功/失败/取消）
4. 条件激活：按钮何时可点、何时灰化、激活条件是什么
5. 异常处理：错误提示文案、重试机制、冷却时间、锁定规则
6. 空状态：无数据时展示什么图标和文案
7. 数据规则：校验规则、字符限制、格式要求

**禁止写入**：

- 色值（如 `#007AFF`）、字号（如 `14px`）、圆角（如 `12px`）、间距像素等设计规格
- 这些属于设计稿的职责，不是 PRD 的内容

**示例**（登录页中一个按钮的描述粒度）：

```
**3.【Apple Sign-In 按钮】**<br>
　3.1 样式：系统标准黑色圆角按钮，文案 `Continue with Apple`，左侧 Apple Logo 白色图标<br>
　3.2 点击后：弹出 iOS 原生 Apple Sign-In 半屏面板（非页面跳转），Face ID / Touch ID 确认<br>
　3.3 授权结果<br>
　　3.3.1 成功 → 后端收到 Apple ID Token（含 email、name）<br>
　　　a. 数据库无此用户 → 创建新账号（昵称=Apple name，头像=首字母色块）→ 跳转首页 Feed Tab<br>
　　　b. 数据库已有此用户 → 直接登录 → 跳转首页 Feed Tab<br>
　　3.3.2 取消 → 面板收起，返回登录页，无提示<br>
　3.4 隐藏邮箱：用户可勾选"隐藏我的邮箱"，系统使用 Apple 中继邮箱
```
