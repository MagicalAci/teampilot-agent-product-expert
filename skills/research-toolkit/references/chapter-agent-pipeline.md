# 章节 Agent 流水线

这份文档定义成稿阶段的标准流水线。

核心原则：

- 不改现有任务目录结构
- 不再由一个写作 agent 包揽整篇
- 每章都走固定流水线
- 图证和正文同等重要

## 总体流程

```text
章节合同 agent
  -> 章节撰写 agent
  -> 配图 agent
  -> 章节事实校验 agent
  -> 下一章

全部章节完成
  -> 总编合稿 agent
  -> 终审 agent
```

## 角色定义

### 1. 章节合同 agent

负责把本章写作边界写进 `06-writing/WRITING_PLAN.md`。

每章至少定义：

- 本章要回答什么问题
- 本章允许下什么判断
- 本章主证据
- 本章必备表格
- 本章必备截图 / 图表
- 本章推荐版式
- 本章是否必须带证据角标
- 本章阻塞项
- 本章完成标准

### 2. 章节撰写 agent

只负责正文和表格，不临时改章节职责。

必须完成：

- 本节主证据
- 核心判断
- 表格
- 图位需求
- 证据角标或明确章节回指

### 3. 配图 agent

先选真实产品图，再考虑结构图。

负责：

- 选择截图 / 抽帧 / 搜索页 / 节点页
- 判断版式：对比板 / 路径板 / 角色板 / 节点板 / 结构图
- 写中文图注
- 插图到正文
- 回填 `08-visuals/VISUAL_PLAN.md`

### 4. 章节事实校验 agent

负责检查：

- 判断是否超证据
- 角标是否齐
- 图和判断是否一一对应
- 图片路径是否在当前任务目录
- 是否仍有英文图注、坏图、明显留白问题

### 5. 总编合稿 agent

在所有章节通过后统一处理：

- 口径统一
- 图号统一
- 表格风格统一
- 重复内容去重
- `1. 总结` 回写
- `10. 证据附录与索引` 汇编

### 6. 终审 agent

负责全篇最终放行。

必须检查：

- 各章是否都通过本章事实校验
- 三轮事实核查是否完整
- 图证覆盖是否达标
- 是否达到 benchmark 水平

## 各章节的最低要求

| 章节 | 必备表格 | 必备配图 | 角标要求 |
|---|---|---|---|
| `2. 分析建模` | 分析模块表 | 建模图 | 必须 |
| `3. 产品分析` | 功能矩阵 / 路径表 | 对比板或路径板 + 结构图 | 必须 |
| `4. 用户分析` | 用户画像 / JTBD | 角色板或旅程图 | 必须 |
| `5. 社媒分析` | 平台角色 / 主题矩阵 | 可选，按证据需要 | 必须 |
| `6. SEO 与内容分析` | 关键词承接表 | 搜索承接图或搜索页截图 | 必须 |
| `7. 商业化分析` | 定价 / 权益边界表 | 节点板或商业化路径图 | 必须 |
| `8. 竞争判断` | 强弱 / 跟避绕观察表 | 判断矩阵图优先 | 必须 |
| `9. 对我方建议` | 行动建议表 | 一般不强制 | 必须，且回指前文 |

## 商业化章节的特别说明

`7. 商业化分析` 不能只看公开定价口径。

只要任务目录里已有这些图，就必须优先下沉：

- 价格页
- 会员页
- 权益页
- 认证页
- 解锁页
- 打印 / 导出 / 下载节点

这些图应该进入本章正文，而不是只停留在素材目录或附录。

## 这套流水线和目录的关系

这套流水线只改变“如何写”，不改变“放哪里”。

也就是说：

- 任务目录仍然用当前的 `00-admin` 到 `08-visuals`
- Skill 对外分发仍然可以用 `assets / examples / references / scripts`
- 章节 agent 是角色与合同拆分，不是目录拆分
