# 调研审核门禁协议

这份协议定义两个审核门禁：

1. `采集门禁`：搜索阶段后能否进入体验与写作
2. `成稿门禁`：正文与图证完成后能否作为最终成品放行

核心原则：

- 搜索子代理跑完后，不能直接进入正式报告
- 必须先经过独立审核代理
- 不合格就定向打回
- 最多允许 5 轮
- 采集门禁通过，不等于成稿门禁也通过

## 1. 审核代理

默认使用：

- `research-review`

它负责检查：

- 子代理是否都已执行
- 每个平台标准产物是否存在
- 样本量是否达标，或是否已取得用户豁免
- 是否越级写成正式总报告口径
- 是否缺真实体验、写作规划、事实核查等后续阶段
- 成稿阶段是否完成章节配图、章节事实校验、三轮核查和终稿级图证检查
- 对小红书这类平台，是否同时具备帖子层和评论层，而不是只有帖子池

说明：

- 子代理 `summary.md` 允许写平台级洞察、候选判断、策划启发
- 审核代理不应把“有价值的平台洞察”误判为偷跑
- 只有当子代理把单平台内容直接写成跨章节总判断、正式结论或最终策略时，才算越级

## 2. 采集门禁

### 通过

- 允许进入“核对采集结果”阶段
- 还不允许直接写正式报告

### 打回

审核代理必须写明：

- 打回给哪个子代理
- 哪个平台不合格
- 缺什么
- 应补多少
- 补完后重新交给谁复核

## 3. 成稿门禁

成稿门禁在以下条件满足后才允许放行：

- `WRITING_PLAN.md` 中各章状态完整
- 每章都走过 `章节合同 -> 章节撰写 -> 配图 -> 章节事实校验`
- `07-factcheck/` 三轮核查完整
- `08-visuals/VISUAL_PLAN.md` 已回填
- 正文图证已覆盖关键章节

成稿门禁至少检查：

- 产品、用户、商业化、竞争这几章是否遗漏应下沉的真实产品图
- 图片是否来自当前任务目录
- 图注是否中文
- 图和章节判断是否一一对应
- 是否仍有英文 SVG 标签、坏图、空图或明显无意义留白
- 总结、正文、附录之间是否回指一致
- 若正文引用“小红书评论区反馈”，必须能追到评论层数据，而不是只追到帖子正文

### 成稿放行

- 允许作为 benchmark 或最终交付物
- 允许被纳入案例库

### 成稿打回

必须写明：

- 打回哪一章
- 缺什么图证或表格
- 是章节撰写问题、配图问题还是事实校验问题
- 返工后重新交给谁复核

## 4. 5 轮规则

审核循环最多 5 轮：

1. 审核
2. 打回
3. 子代理补做
4. 重新审核

如果第 5 轮后仍不达标：

- 不再强行放行
- 必须停下来向用户汇报真实问题
- 由用户决定是否降标准、改范围或终止

## 5. 明确禁止

- 没过审核就写正式报告
- 样本量不足却默认放行
- 缺体验素材却写“深度体验结论”
- 审核代理只给一句“通过/不通过”，不写原因
- 把平台级洞察一律当成违规内容
- 采集门禁通过后，跳过成稿门禁直接对外交付
- 明明有真实产品图，却只用抽象结构图代替
- 小红书只有帖子层、没有评论层，却默认放行为高标准样本
