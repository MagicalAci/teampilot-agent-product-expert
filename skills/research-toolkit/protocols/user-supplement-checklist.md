# 用户补料清单

这份清单用于定义：

- 哪些情况下必须请用户补材料
- 用户应该补什么
- 补到什么程度才算可用
- 补来的材料如何进入共享证据库

目标不是“多要材料”，而是“只要真正影响判断的材料”。

## 1. 什么时候要补料

出现以下任一情况，就不要硬推断，直接进入补料流程：

| 场景 | 说明 | 对哪一阶段影响最大 |
|---|---|---|
| 原生 App 才有核心价值 | 网页端信息不足以判断产品真实体验 | 产品分析、用户分析（含旅程） |
| 登录后才出现关键模块 | 公开页面只能看到外层壳 | 产品分析、用户分析 |
| 会员/付费后才展示核心能力 | 无法判断商业化结构和价值兑现点 | 商业化分析、竞争判断 |
| 公开评论太少或噪音太大 | 难以支撑用户分层和传播判断 | 用户分析（含旅程）、社媒分析 |
| SEO/内容承接依赖站内深层页面 | 公开列表页不够判断内容结构 | SEO 与内容分析 |
| 用户行为链路需要动态观察 | 静态截图不能说明交互与反馈 | 用户分析（含旅程）、产品分析 |

## 2. 补料优先级

优先级规则：

1. 先补会改变结论的材料
2. 再补能增强可信度的材料
3. 最后补用于展示的材料

| 优先级 | 材料类型 | 为什么优先 |
|---|---|---|
| P0 | 登录后首页、核心功能页、会员页、支付前页、主路径录屏 | 直接决定产品、商业化、旅程判断 |
| P1 | 应用内通知、推荐页、个人中心、设置页、历史记录页 | 用于补结构完整性和留存机制 |
| P2 | 宣传海报、品牌视觉、运营活动图 | 更多服务于展示，不直接支撑核心判断 |

## 3. 各类材料的补充标准

## 3.1 截图

适用：

- 登录后页面
- 会员页
- 支付页
- 原生 App 关键页面

最低要求：

| 项目 | 要求 |
|---|---|
| 清晰度 | 页面文字可读，不要模糊压缩图 |
| 完整度 | 尽量包含顶部导航、主体内容、底部按钮 |
| 数量 | 每个关键页面至少 1 张 |
| 说明 | 每张图都要说明“这是什么页”和“它在流程里的位置” |

推荐命名：

```text
产品名-page-页面名称-日期.png
例：duolingo-page-paywall-2026-03-09.png
```

推荐配套说明模板：

```markdown
页面名称：
所在阶段：认知 / 注册 / onboarding / 使用 / 付费 / 留存
页面作用：
为什么这页重要：
是否需要我重点关注某个区域：是 / 否
```

## 3.2 录屏

适用：

- onboarding 流程
- 首次使用主路径
- 付费转化路径
- 复杂交互流程

最低要求：

| 项目 | 要求 |
|---|---|
| 时长 | 30 秒到 3 分钟为宜 |
| 起点 | 从进入页面前一刻开始 |
| 终点 | 到用户获得结果或被阻断为止 |
| 节奏 | 正常操作速度，不要快放 |
| 旁白 | 可选，但建议说明你为什么这么点 |

推荐命名：

```text
产品名-flow-流程名称-日期.mp4
例：duolingo-flow-first-lesson-2026-03-09.mp4
```

推荐流程标签：

- `首次进入`
- `注册转化`
- `首次完成核心任务`
- `免费转付费`
- `留存回访`

## 3.3 登录后页面

最有价值的补料类型之一。

至少补以下 4 类中的 2-3 类：

| 页面 | 用途 | 常影响的判断 |
|---|---|---|
| 登录后首页 | 看信息架构和主承接逻辑 | 产品结构、留存设计 |
| 核心功能页 | 看价值兑现方式 | 产品分析、竞争判断 |
| 历史/结果页 | 看长期价值和留存抓手 | 用户分析（含旅程）、留存判断 |
| 个人中心/设置页 | 看账号体系、会员体系、权限设计 | 商业化分析、用户判断 |

## 3.4 付费页与商业化页面

如果公开定价不足，至少补：

| 页面 | 必要性 | 用途 |
|---|---|---|
| 定价页 | 必须 | 看套餐、计费单位、价格锚点 |
| 权益对比页 | 强烈建议 | 看免费与付费边界 |
| 支付前确认页 | 强烈建议 | 看转化文案、限时刺激、默认选项 |
| 试用结束页 | 可选但很有价值 | 看转付费时机和压迫感设计 |

这类页面最好附带一句：

```markdown
这个页面是在什么动作之后出现的？
```

## 3.5 一手用户反馈

当公开评论不足时，这部分价值极高。

可接受材料：

- 访谈摘要
- 客服记录脱敏版
- 用户群聊天节选
- 内部调研笔记
- 观察记录

最低要求：

| 项目 | 要求 |
|---|---|
| 条数 | 至少 5 条有效反馈 |
| 真实性 | 来自真实使用者或真实潜在用户 |
| 脱敏 | 去除姓名、手机号、订单号等隐私信息 |
| 结构 | 最好带上用户身份、场景、原话、问题 |

推荐整理模板：

```markdown
用户身份：
使用场景：
原话：
暴露的问题：
你主观判断这条反馈是否典型：高 / 中 / 低
```

## 3.6 SEO 与内容相关页面

当站点 SEO 是重要维度时，建议补：

| 材料 | 用途 |
|---|---|
| 博客列表页截图 | 看栏目结构和更新频率 |
| 某篇高流量文章截图或链接 | 看承接方式和转化设计 |
| 搜索结果页截图 | 看品牌词、产品词和竞品词表现 |
| 内链结构说明 | 看内容如何导向注册、试用或转化 |

## 4. 用户补料时必须一并说明的信息

无论给截图、录屏还是访谈，都尽量同时给下面这些信息：

| 字段 | 是否必须 | 说明 |
|---|---|---|
| 材料名称 | 必须 | 用于索引 |
| 产品名 | 必须 | 避免混入别的产品 |
| 获取日期 | 必须 | 用于判断时效性 |
| 页面/流程所处阶段 | 必须 | 认知、注册、使用、付费、留存 |
| 你是怎么进入这个页面的 | 强烈建议 | 帮助还原路径 |
| 是否为会员态/登录态 | 必须 | 影响判断前提 |
| 你觉得这份材料最值得我看什么 | 可选 | 便于聚焦 |

## 5. AI 收到补料后的处理规则

收到补料后，应立即做三件事：

1. 标记这份材料属于哪个证据池
2. 标记它解决了哪条“不确定性”
3. 更新对应章节的结论可信度

推荐记录格式：

| 材料 | 证据池 | 解决的问题 | 影响章节 | 可信度变化 |
|---|---|---|---|---|
| `duolingo-page-paywall-2026-03-09.png` | 定价与商业化池 | 补足会员权益和支付前文案 | 商业化分析、竞争判断 | 中 -> 高 |

## 6. 向用户要材料时的标准话术

## 6.1 产品深层体验缺失

```markdown
我已经完成公开资料分析，但这个产品的关键价值更可能出现在登录后或 App 内。

为了避免我凭公开页面硬猜，请你补下面任意 2-3 项：
1. 登录后首页截图
2. 核心功能页截图
3. 首次完成核心任务的录屏
4. 历史记录或结果页截图
```

## 6.2 商业化链路缺失

```markdown
我已经拿到基础定价信息，但还缺少真正决定转化判断的页面。

请优先补：
1. 会员权益页
2. 支付前确认页
3. 试用结束后的转付费页
4. 你是通过什么操作进入这些页面的
```

## 6.3 用户反馈不足

```markdown
公开评论足够支撑基础结论，但还不足以支撑高置信度的人群判断。

如果你手里有一手材料，请补任意一种：
1. 访谈摘要
2. 客服记录脱敏版
3. 用户群讨论节选
4. 内部观察笔记
```

## 7. 不要补什么

以下材料通常价值不高，除非有明确用途，否则不要优先提供：

- 模糊截图
- 无上下文的单张界面图
- 纯宣传海报
- 无日期的旧截图
- 无来源说明的二手截图
- 无脱敏处理的隐私信息

## 8. 一句话原则

补料的目标不是把材料堆满，而是让 `关键结论从推断升级为可证实判断`。
