# 可视化代理协议

可视化不是正文最后的装饰环节，而是章节流水线中的固定一步。

默认顺序：

1. 章节正文
2. 章节配图
3. 章节事实校验
4. 全文事实核查
5. 全文结构图补齐

说明：

- 有真实产品图时，先用真实产品图
- 真实图不够解释结构时，再补 SVG / PNG 结构图
- 不允许用结构图替代本来就已经拿到的真实界面图

目标不是“加很多图”，而是只补关键图，帮助结构化理解。

## 优先考虑的图

- 端口对比板
- 重点功能路径板
- 用户角色板
- 商业化 / 治理节点板
- 分析建模图
- 产品结构图
- 用户分层图
- 社媒主题聚类图
- SEO 内容结构图
- 商业化路径图
- 竞争判断矩阵

## 输出要求

- 真实截图优先，SVG 次之
- 必须可插入 Markdown 文档
- 图和结论一一对应
- 图不能替代正文，只做结构化辅助
- 每张图都要有中文图注
- 每张图都要能回答“这张图证明什么”
- 图片路径必须来自当前任务目录
- 图板应优先使用统一版式，可参考 `assets/chapter-visual-blocks.md`

## 章节触发规则

### 产品分析

- 已有真实截图时，必须至少补一组 `对比板` 或 `路径板`

### 用户分析

- 界面能明显对应不同角色时，必须补 `角色板`

### SEO 与内容分析

- 已有搜索页、落地页截图时，优先补 `搜索承接板`

### 商业化分析

- 已有价格页、认证页、会员页、权益页、解锁页时，必须补 `节点板`

### 竞争判断

- 若需要压缩判断，优先补矩阵图或强弱对照图

## 成稿级视觉检查

终稿前至少检查：

- 是否还有应该下沉但未下沉的真实产品图
- 是否存在英文图注、英文 SVG 标签或英文占位
- 是否存在无意义大面积留白
- 图和图之间的密度、宽度、边框、图注是否统一
- 图号、图注和正文引用是否一致

## 输出文件

```text
08-visuals/svg/
08-visuals/png/
08-visuals/VISUAL_PLAN.md
```
