# 单产品深度分析

## 适用场景

- 对某一竞品或目标产品做全方位深度拆解
- 需要从产品、用户、社媒、SEO、商业化等多维度形成完整判断
- 典型触发：`/竞品分析 [产品名]`

## 章节结构

| 序号 | 章节名 | 核心目标 | 主要证据来源 | 必备表格/图表 |
|------|--------|----------|-------------|--------------|
| 1 | 总结 | 全篇核心判断浓缩，最后回写 | 各章结论汇总 | 一页纸摘要表 |
| 2 | 分析建模 | 搭建分析框架，定义每个模块的判断目标 | 产品公开信息 + 用户预设 | 分析模块表、建模图 |
| 3 | 产品分析 | 拆解产品架构、核心功能、关键路径 | App/Web 实测截图 | 产品架构图、功能矩阵、路径板 |
| 4 | 用户分析 | 还原用户画像、旅程、JTBD | 社媒帖子 + 评论 + 官方描述 | 用户画像表、旅程图、JTBD 表 |
| 5 | 社媒分析 | 提取正向/负向主题，定位核心口碑 | 小红书/知乎/微博/B站 | 平台角色分工表、正负主题矩阵 |
| 6 | SEO 与内容分析 | 判断内容策略和搜索承接能力 | 搜索引擎 + 官网内容 | 关键词承接表、内容格局图 |
| 7 | 商业化分析 | 拆解定价、权益、付费节点 | 价格页/会员页截图 | 定价总表、权益边界表、节点板 |
| 8 | 竞争判断 | 给出强项/短板/护城河/脆弱点 | 全篇证据交叉 | 强项短板表、判断矩阵图 |
| 9 | 对我方建议 | P0/P1/P2 行动建议，回指前文证据 | 各章结论 | 行动建议表 |
| 10 | 证据附录 | 全部证据索引、截图目录、数据源清单 | 采集产物 | 证据索引表、截图索引表 |

## 写作顺序

实际写作按 2→3→4→5→6→7→8→9→1→10 推进，总结最后回写。

每章走完整流水线（不可缩减）：

1. **章节合同**：回填 WRITING_PLAN.md，确定判断边界、证据、表格、配图、阻塞项
2. **章节撰写**：正文 + 表格 + 证据角标 + 图位需求
3. **配图**：优先真实截图/抽帧，再补结构图；图注必须中文且说明判断点
4. **章节事实校验**：检查强判断、证据角标、图文对应、路径、留白

全篇通过后交给总编合稿统一口径、图号、重复段落，再回写总结和证据附录。

### 章节依赖

- 第 3 章（产品分析）依赖体验截图完成
- 第 5 章（社媒分析）依赖采集审核通过
- 第 8 章（竞争判断）依赖 3-7 章全部完成
- 第 9 章（对我方建议）依赖第 8 章结论
- 第 1 章（总结）和第 10 章（附录）最后统一补写

## 体验要求

- **App 体验**：必须完成完整用户流程（注册→核心功能→付费节点），留存截图/录屏
- **Web 体验**：必须遍历主要页面（首页→功能页→价格页→内容页），留存截图
- 体验产出的截图必须下沉到对应章节正文，不能只放附录

## 采集要求

| 维度 | 目标 |
|------|------|
| 社媒采集 | 每平台有效样本 ≥ 200 条（小红书必须区分帖子层和评论层） |
| 搜索引擎 | 核心词 + 长尾词覆盖，至少 50 条有效结果 |
| App Store | 评分、评论（近 6 月）、版本历史 |
| 关键词分类 | 核心词、别名词、功能词、长尾词、评价词、问题词、竞品词 |

## 门禁差异

| 门禁 | 要求 |
|------|------|
| 采集审核 | 搜索阶段需独立审核代理复核，未放行不进入写作 |
| 样本量检查 | 低于 200 条/平台必须回问是否补抓 |
| 事实核查 | 三轮独立核查（章节级 → 全稿级 → 交叉验证），落地轮次记录 |
| 证据可追溯 | 关键判断附可点击证据索引 `[n](路径)` |
| 配图合规 | 真实截图优先，图注必须中文且说明判断点 |

## 交付物

| 产物 | 格式 | 说明 |
|------|------|------|
| 深度分析报告 | Markdown（10 章） | 主交付物，含表格、配图、证据角标 |
| WRITING_PLAN.md | Markdown | 章节合同记录，含每章状态和阻塞项 |
| 证据附录 | Markdown + CSV | 证据索引表 + 结构化数据 |
| 截图目录 | 文件夹 | 按章节组织的产品截图/录屏抽帧 |
| 事实核查记录 | Markdown | 三轮核查的轮次日志 |

## 与其他任务类型的区别

- 与「方向调研」：本类型聚焦单品深度，不做赛道全景扫描
- 与「用户研究」：本类型用户分析只是 10 章之一，不做三层证据穿透
- 与「市场全景」：本类型不做多产品横向对比，只以单品为中心展开
