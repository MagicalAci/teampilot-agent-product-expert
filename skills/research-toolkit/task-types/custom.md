# 自定义调研

## 适用场景

- 用户有自己的分析框架，不适用预设章节结构
- 调研目标特殊（如技术调研、政策调研、供应链调研等）
- 需要灵活定制章节、采样标准和核查深度
- 典型触发：`/自定义调研 [主题]`、用户主动提供分析框架

## 章节结构

由用户在需求澄清阶段定义，AI 协助补全。

### 澄清阶段必须确认

| 确认项 | 说明 | 默认值（用户未指定时） |
|--------|------|----------------------|
| 章节列表 | 每章名称和核心目标 | 无默认，必须用户给出 |
| 每章必备表格 | 至少列出核心表格类型 | 每章至少 1 张表格 |
| 每章必备图表 | 是否需要配图、什么类型 | 真实截图优先 |
| 证据索引要求 | 是否需要可点击角标 | 是 |
| 写作顺序 | 是否有章节依赖关系 | 按序号顺序推进，总结最后 |

### AI 补全规则

用户给出章节骨架后，AI 负责补全以下内容并回给用户确认：

1. 每章的判断边界（允许下什么结论、不允许什么）
2. 每章的主要证据来源
3. 章节间的依赖关系（哪些章节必须先写）
4. 建议增补的章节（如用户遗漏了总结或证据附录）

补全后形成正式章节合同，用户确认后锁定。

### 强制保留章节

无论用户如何自定义，以下两个章节始终保留：

| 章节 | 位置 | 说明 |
|------|------|------|
| 总结 | 最后回写，放在第 1 章位置 | 全篇核心结论浓缩 |
| 证据附录 | 最后一章 | 证据索引、数据源清单 |

## 采集要求

由用户指定，AI 提供建议。

| 参数 | 用户指定 | AI 建议默认值 |
|------|----------|--------------|
| 社媒平台 | 用户选择 | 小红书 + 知乎 |
| 每平台采样量 | 用户指定 | 200 条/平台 |
| 是否区分帖子层/评论层 | 用户指定 | 涉及用户痛点时建议区分 |
| 搜索引擎采集 | 用户指定 | 是，至少 50 条 |
| 体验要求 | 用户指定 | 涉及产品分析时建议实测 |
| DeerFlow 使用 | 用户指定 | 涉及市场数据时建议使用 |

## 门禁差异

| 门禁 | 要求 |
|------|------|
| 章节合同 | 必须在写作前锁定，变更需用户重新确认 |
| 采集审核 | 按用户指定的采样标准执行，默认走独立审核 |
| 样本量检查 | 按用户指定阈值，未指定则默认 200 条/平台 |
| 事实核查 | 默认三轮；用户可降级为两轮（必须明确同意） |
| 章节流水线 | 每章仍走 合同→撰写→配图→事实校验 四步 |
| 配图合规 | 真实截图优先原则不变 |

### 门禁降级规则

用户可申请降级的门禁：

| 可降级项 | 降级条件 | 最低标准 |
|----------|----------|----------|
| 事实核查轮次 | 用户明确同意 | 不低于两轮 |
| 采样量 | 用户明确指定 | 不低于 50 条/平台 |
| 体验环节 | 调研不涉及产品分析 | 可跳过 |

不可降级的门禁：

- 章节合同必须锁定
- 总结和证据附录不可删除
- 章节流水线四步不可缩减
- 证据索引角标不可取消

## 交付物

| 产物 | 格式 | 说明 |
|------|------|------|
| 调研报告 | Markdown（用户定义章节数） | 主交付物 |
| WRITING_PLAN.md | Markdown | 章节合同 + 写作进度 |
| 证据附录 | Markdown + CSV | 证据索引 + 结构化数据 |
| 事实核查记录 | Markdown | 核查轮次日志 |
| 其他（按需） | 用户指定 | 如竞品追踪表、数据集等 |
