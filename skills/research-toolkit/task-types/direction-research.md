# 方向/赛道调研

## 适用场景

- 新产品或新方向启动前的可行性判断
- 需要回答"这个方向值不值得做、怎么切入"
- 需要同时产出分析文档和可持续跟踪的竞品表
- 典型触发：`/方向调研 [赛道名]`、`/赛道调研 [领域]`

## 章节结构

| 序号 | 章节名 | 核心目标 | 主要证据来源 | 必备表格/图表 |
|------|--------|----------|-------------|--------------|
| 1 | 总结 | 方向可行性结论 + Go/No-Go 建议，最后回写 | 各章结论汇总 | 决策摘要表 |
| 2 | 市场格局 | 赛道规模、增长趋势、玩家分层 | DeerFlow 深度研究 + 公开报告 | 市场规模图、玩家分层表 |
| 3 | 目标用户与需求 | 谁在用、为什么用、核心痛点 | 社媒采集 + 搜索引擎 | 用户分群表、需求优先级矩阵 |
| 4 | 竞品全景 | 赛道内所有相关产品的横向扫描 | DeerFlow + App Store + 官网 | 竞品全景矩阵（功能/定价/用户量） |
| 5 | 重点竞品深拆 | 3-5 个核心竞品的产品/商业/技术拆解 | 官网体验 + 社媒 + 公开数据 | 每个竞品的功能表 + 强弱项表 |
| 6 | 需求真实性验证 | 用多源证据交叉验证需求是否真实存在 | 社媒原话 + 搜索量 + 评论 | 需求验证矩阵（证据源×需求项） |
| 7 | 机会与风险 | SWOT 视角下的机会窗口和潜在风险 | 全篇证据交叉 | SWOT 表、风险等级表 |
| 8 | 行动建议 | 切入策略、MVP 范围、优先级路线 | 各章结论 | 行动路线表（P0/P1/P2） |

## 写作顺序

按 2→3→4→5→6→7→8→1 推进。第 5 章可拆为多个子节（每个竞品一节）并行准备素材，但落文仍逐节推进。

每章走完整流水线：章节合同 → 撰写 → 配图 → 事实校验。

### 章节依赖

- 第 3 章（目标用户）依赖社媒采集完成
- 第 4 章（竞品全景）依赖 DeerFlow 产出 + 搜索采集
- 第 5 章（深拆）依赖第 4 章确定重点竞品名单 + 体验截图
- 第 6 章（需求验证）依赖第 3、4、5 章的证据积累
- 第 1 章（总结）最后回写

### 双交付协调

分析调研文档和竞品追踪表并行维护：
- 写第 4、5 章时同步填充追踪表的功能/定价/融资字段
- 追踪表在报告交付后仍可持续更新

## 核心工具

- **DeerFlow 深度研究**：市场格局、行业报告、融资数据的主要来源
- **社媒采集**：用户需求和痛点的一手证据
- **App Store / 官网体验**：竞品功能和定价的直接来源

## 采集要求

| 维度 | 目标 |
|------|------|
| 社媒采集 | 每平台有效样本 ≥ 200 条 |
| DeerFlow 研究 | 至少覆盖市场规模、融资、玩家清单三个维度 |
| 竞品体验 | 重点竞品（3-5 个）需完成核心流程体验 + 截图 |
| 搜索引擎 | 赛道关键词 + 需求词 + 竞品词，至少 80 条有效结果 |

## 门禁差异

| 门禁 | 要求 |
|------|------|
| 采集审核 | DeerFlow 产出需人工确认数据时效性 |
| 样本量检查 | 低于 200 条/平台必须补抓 |
| 事实核查 | 三轮：章节级 → 全稿级 → 需求真实性专项核查 |
| 竞品体验 | 重点竞品必须实测，不能只靠二手描述 |
| 需求验证 | 第 6 章必须有 ≥ 3 个独立证据源交叉确认 |

## 交付物

| 产物 | 格式 | 说明 |
|------|------|------|
| 方向调研报告 | Markdown（8 章） | 主交付物，含市场数据、竞品拆解、行动建议 |
| 竞品追踪表 | CSV / Excel | 可持续更新的结构化竞品数据，含功能/定价/融资字段 |
| WRITING_PLAN.md | Markdown | 章节合同 + 写作进度 |
| 截图目录 | 文件夹 | 重点竞品的体验截图 |
| 事实核查记录 | Markdown | 三轮核查轮次日志 |

## 竞品追踪表字段规范

追踪表至少包含以下字段：

| 字段 | 类型 | 说明 |
|------|------|------|
| 产品名 | 文本 | 产品全称 + 常用简称 |
| 官网 | URL | 主站地址 |
| 核心功能 | 文本 | 3-5 个核心功能关键词 |
| 目标用户 | 文本 | 主要用户群 |
| 定价模式 | 文本 | 免费/付费/订阅/混合 |
| 融资阶段 | 文本 | 天使/A/B/已盈利/未知 |
| 月活估算 | 数值 | 公开数据或推测（标注来源） |
| 采集日期 | 日期 | 数据时效性标注 |

## 与其他任务类型的区别

- 与「单产品深度」：本类型覆盖 3-5 个竞品但每个不深入到 10 章级别
- 与「市场全景」：本类型重可行性判断和行动建议，不重横向对比矩阵
