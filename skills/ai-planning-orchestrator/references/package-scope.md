# 打包范围

## 开发最小接入包应包含

### 核心源码

- 核心生成器 / 编排器
- 模型客户端
- route / API handler 示例
- route / handler 测试

### 运行与验证

- demo 脚本
- 输入样例
- 输出样例
- 单元测试
- AI测试报告

### 文档

- AI PRD
- Prompt 文档
- README / 调用文档
- 脚本包清单

## README 至少说明

- 入口文件在哪
- 用什么命令验证
- 当前平台配置是什么
- `thinking` 是否开启
- 当前模型对结构化输出的实际支持情况
- CLI / HTTP 调用方式是什么
- 输入样例和输出样例分别在哪
- 哪些能力由目标项目继续补

## 命令对应关系

- `/AI脚本` 默认产出脚本、样例、README
- `/AI脚本` 还应补齐 README / 调用文档 与脚本包清单
- `/AI测试` 默认产出 AI测试报告
- `/AI策划` 在需要开发交接时，才继续打最小接入包

## 不要漏掉的边界说明

- 鉴权
- CORS
- 域名 / 网关放行
- 日志和监控
- 正式生产环境的 Key 注入方式
