# 评审清单

## 命令定位

- [ ] `README` 和 `SKILL.md` 已明确四条主命令：`/AI策划`、`/AIPRD`、`/AI脚本`、`/AI测试`
- [ ] 全流程、PRD、脚本、测试四种模式的边界清楚，不互相混淆
- [ ] 旧命令如果保留，只作为兼容说明，不抢主入口

## AI PRD

- [ ] 顶部元信息完整：时间、平台、版本号、撰写人
- [ ] 章节顺序严格是 `输入数据结构定义 -> AI策略 -> 提示词模块 -> AI脚本`
- [ ] 输入结构部分写清总结构、一级对象、数组项、回退口径
- [ ] AI 策略写清生成顺序、平台配置、业务红线
- [ ] 模块策略写清候选池、排序公式、去重规则、字段回退口径
- [ ] 提示词模块表格包含 `模块 / 输入参数 / 输出参数 / 输出示例 / 前端映射 / prompt详情`
- [ ] Prompt 正文统一为 `人设 / 任务 / 规则 / 输出规范 / 严格禁止 / 思考链`
- [ ] AI脚本部分包含测试 JSON、运行入口、输出示例、压缩包说明

## AI脚本

- [ ] 已建立在冻结的字段 / Prompt 合同之上
- [ ] 至少包含核心脚本、样例、README / 调用文档
- [ ] 开发调用文档写清 CLI / HTTP / route 的适用入口
- [ ] 脚本包清单已列清哪些文件直接可用、哪些仍需目标项目补
- [ ] 没有在字段未冻结时直接落代码

## AI测试报告

- [ ] 明确区分单元测试、真实调用、兼容性验证
- [ ] 不只贴日志路径，而是有测试结论
- [ ] 写明当前模型是否支持 `json_schema` 或其他结构化输出方式
- [ ] 写明失败时是否有降级策略
- [ ] 写明剩余风险和限制

## 压缩包

- [ ] 至少包含生成器、客户端、handler 示例、测试、样例、Prompt、README / 调用文档
- [ ] README 说明"哪些可以直接用，哪些要目标项目自己补"
- [ ] 已提供脚本包清单或等价文件列表
- [ ] 平台配置与文档一致
- [ ] 输出示例与 PRD 字段一致

## 一致性

- [ ] 文档里的模型名、APPID、agent ID、chat_url 与代码一致
- [ ] 文档里的输出字段与示例 JSON 一致
- [ ] 文档里的验证结论与实际测试结果一致
- [ ] Prompt 文档、AI PRD、脚本输出没有互相打架
