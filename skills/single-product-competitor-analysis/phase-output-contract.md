# 阶段产出标准

这份文档定义三条指令的阶段产物：

- `/竞品分析`
- `/爬取`
- `/竞品引导`

核心原则：

- 关键讨论由 AI 归纳沉淀，不由脚本自动编造
- 平台数据和体验素材必须落盘
- 脚本负责机械劳动，AI 负责提问、洞察、缺口判断和正文交付

## 1. 总览表

| 指令 | 阶段 | 核心目标 | 必须输出 | 是否要用户确认 |
|---|---|---|---|---|
| `/竞品分析` | 需求澄清 | 锁定边界与目标 | 任务卡草案 | 是 |
| `/竞品分析` | 任务初始化 | 建独立任务目录和模板 | 任务骨架 | 否 |
| `/竞品分析` | 关键词核对 | 明确搜索词和长尾词 | `SEARCH_KEYWORDS.md` `SEARCH_KEYWORDS.csv` | 是 |
| `/竞品分析` | 子代理采集 | 建立共享证据库 | 子代理清单 + 平台 `summary.md + data.csv` | 低样本时要 |
| `/竞品分析` | 审核门禁 | 审查采集是否达标 | 审核结果 / 打回单 | 是 |
| `/竞品分析` | 真实体验 | 拿到截图、录屏、洞察 | `04-experience/*` | 是 |
| `/竞品分析` | 写作共创 | 先建模再逐章写 | `WRITING_PLAN.md` + 章节草稿 | 是 |
| `/竞品分析` | 核查补图 | 事实核查与 SVG 图 | `07-factcheck/*` `08-visuals/*` | 建议确认 |
| `/爬取` | 采集澄清 | 明确平台、关键词、阈值 | 采集任务卡 | 是 |
| `/爬取` | 平台采集 | 产出平台数据与 AI 总结 | 平台 `summary.md + data.csv` | 低样本时要 |
| `/竞品引导` | 体验规划 | 规划体验路径与记录方式 | `EXPERIENCE_SCRIPT.md` | 是 |
| `/竞品引导` | 素材沉淀 | 存截图、录屏、关键帧、洞察 | `04-experience/*` | 是 |

## 2. `/竞品分析` 的阶段要求

### 2.1 需求澄清

默认规则：

- 默认完整章节顺序推进，不问“先看哪个模块”
- 默认纳入 App 深度体验
- 默认按最高标准交付，不问“给谁看”

最少要确认：

- 产品名
- 主链接或入口
- 分析目标
- 这轮重点看什么
- 是否有特殊限制、禁区或必须补看的角度

可延后确认：

- 当前我方产品或业务是什么

说明：

- 如果要写“对我方建议”而上下文里还没有我方背景，可在写作前再补问
- 不要让这些信息阻塞任务启动

必须落盘：

- `00-admin/TASK_CARD.md`
- `01-process/PROCESS_LOG.md`
- `01-process/KEY_DECISIONS.md`

### 2.2 任务初始化

必须生成：

```text
00-admin/
01-process/
02-keywords/
03-platforms/
04-experience/
05-synthesis/
06-writing/
07-factcheck/
08-visuals/
```

默认规则：

- 每次 `/竞品分析` 都新建一个独立任务目录
- 不默认读取同产品上一次任务的过程文档、决策文档和主报告
- 只有用户明确要求继续/引用旧任务时，才允许挂接旧目录

### 2.3 关键词核对

必须覆盖：

- 核心词
- 别名词
- 功能词
- 长尾词
- 评价词
- 问题词
- 竞品词

必须输出：

- `02-keywords/SEARCH_KEYWORDS.md`
- `02-keywords/SEARCH_KEYWORDS.csv`

### 2.4 子代理采集

要求：

- 先列子代理清单
- 明确每个子代理调研什么平台
- 按批次并发，单批最多 4 个

必须输出：

- `05-synthesis/SUBAGENT_ROSTER.md`

每个搜索子代理必须输出：

- `03-platforms/<platform>/summary.md`
- `03-platforms/<platform>/data.csv`

默认平台：

- `web`
- `appstore`
- `xiaohongshu`
- `weibo`
- `bilibili`
- `zhihu`
- `seo`
- `pricing`

样本量规则：

- 默认至少 `200` 条/篇有效样本
- 不足时必须回问用户

### 2.5 审核门禁

要求：

- 搜索子代理完成后，必须交给独立审核代理
- 不合格要定向打回
- 最多允许 5 轮

必须输出：

- `05-synthesis/REVIEW_GATE.md`
- 审核结果
- 打回原因
- 指定返工的子代理

完成条件：

- 审核代理明确放行
- 否则不能进入真实体验或正式报告

### 2.6 真实体验

必须输出：

- `04-experience/uploads/`
- `04-experience/screenshots/`
- `04-experience/frames/`
- `04-experience/notes/`
- `04-experience/EXPERIENCE_SCRIPT.md`
- `04-experience/EXPERIENCE_REPORT.md`

完成条件：

- 用户完成关键路径体验
- 关键素材已落盘
- 用户洞察已记录
- `EXPERIENCE_REPORT.md` 已说明这些素材将进入哪些章节
- 若存在待补素材，必须标明它会阻塞哪一章写作

### 2.7 写作共创

写作顺序固定为：

1. 分析建模
2. 产品分析
3. 用户分析
4. 社媒分析
5. SEO 与内容分析
6. 商业化分析
7. 竞争判断
8. 对我方建议
9. 总结
10. 证据附录

必须输出：

- `06-writing/WRITING_PLAN.md`
- 各章讨论结果
- 最终 Markdown 草稿
- 当前任务目录内主报告：`06-writing/<task-slug>.md`
- 每章的章节合同状态
- 每章的配图状态
- 每章的章节事实校验状态

`WRITING_PLAN.md` 至少要为每章写清：

- 本章问题
- 本章核心判断
- 本章主证据
- 本章必备表格
- 本章必备截图 / 图表
- 本章推荐版式
- 本章是否需要证据角标
- 本章阻塞项
- 本章完成标准

每章固定流水线：

1. 章节合同
2. 章节撰写
3. 配图
4. 章节事实校验

完成条件：

- 章节正文已写
- 章节表格已齐
- 本章应下沉的真实图片已下沉
- 图注与角标已补齐
- 章节事实校验已通过

### 2.8 核查与补图

必须输出：

- `07-factcheck/FACTCHECK_PLAN.md`
- `07-factcheck/round-1.md`
- `07-factcheck/round-2.md`
- `07-factcheck/round-3.md`
- `07-factcheck/final-status.md`
- `08-visuals/svg/`
- `08-visuals/png/`
- `08-visuals/VISUAL_PLAN.md`

执行规则：

- 三轮核查之前，应先完成各章的章节事实校验
- 三轮核查都必须调用独立只读 `research-review` 子代理
- 不允许主代理手动降级代审
- Round 3 必须检查主报告路径仍在当前任务目录内
- 成稿放行前还要检查图证覆盖、图文对应、中文图注和本地素材路径

## 3. `/爬取` 的阶段要求

### 3.1 挂载规则

- 默认创建轻量采集任务
- 只有用户明确要求挂到当前任务或继续某任务时，才允许复用旧目录

### 3.2 最少要确认

- 抓什么主题或产品
- 抓哪些平台
- 为什么抓
- 默认样本量是否要提高或降低

### 3.3 必须输出

- 采集任务卡
- 子代理清单
- 关键词清单
- 每个平台的 `summary.md`
- 每个平台的 `data.csv`
- AI 的平台总结、平台洞察、缺口和下一步建议
- 审核结果

### 3.4 完成条件

- 所有指定平台都有标准产物
- 样本量不足的平台已明确处理方式
- AI 已说明哪些证据可直接进入后续分析，哪些仍不够
- 审核代理已放行，或已明确停在第几轮

## 4. `/竞品引导` 的阶段要求

### 4.1 挂载规则

- 默认创建轻量体验任务
- 只有用户明确要求挂到当前任务或继续某任务时，才允许复用旧目录

### 4.2 最少要确认

- 体验目标是什么
- 优先体验哪些路径
- 哪些页面最值得录屏或截图

### 4.3 必须输出

- `04-experience/EXPERIENCE_SCRIPT.md`
- `04-experience/EXPERIENCE_REPORT.md`
- 关键截图
- 原始录屏
- 视频关键帧
- 用户洞察记录

### 4.4 完成条件

- 体验脚本明确
- 素材可复用
- 洞察能进入后续正文
- `EXPERIENCE_REPORT.md` 已回填到可复用判断和待补素材清单
