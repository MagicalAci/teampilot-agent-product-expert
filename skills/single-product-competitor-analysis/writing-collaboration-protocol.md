# 写作共创协议

资料准备完成后，不直接一口气出完整正文。

必须先规划，再逐章共写。

## 1. 写作顺序

1. 分析建模
2. 产品分析
3. 用户分析
4. 社媒分析
5. SEO 与内容分析
6. 商业化分析
7. 竞争判断
8. 对我方建议
9. 总结
10. 证据附录与索引

说明：

- 上面是**实际写作顺序**
- `总结` 只能最后回写
- 不允许先一口气出完整正文，再回头修章节

## 2. 每一章都要共创

进入任何章节前，AI 都要和用户核对：

- 这章要回答什么问题
- 这章想得到什么结论
- 用户的洞察是什么
- AI 的洞察是什么
- 哪些地方证据还不够

每一章落文时，都要显式补：

- `本节主证据`：使用 `[1](相对路径)` 这种可点击索引
- `关键数据表`
- `本章结论`
- `还缺什么`

## 3. 分析建模优先

第 2 章固定是“分析建模”，不是“执行摘要”。

这一章要先搭出：

- 本次分析看哪些模块
- 每个模块为什么重要
- 每个模块最终想得到什么结论

## 4. 逐章输出规则

写作计划必须是逐章推进，不是一次性全出。

至少要做到：

1. `WRITING_PLAN.md` 里写清每章状态、目的、主证据、必备表格和图
2. `WRITING_PLAN.md` 同时承担章节合同作用，至少写清：
   - 本章要回答什么问题
   - 本章允许下什么强判断
   - 本章必备表格
   - 本章必备截图 / 图表
   - 本章推荐版式（对比板 / 路径板 / 角色板 / 节点板 / 结构图）
   - 本章是否必须带证据索引角标
   - 本章阻塞项和完成标准
3. 每次推进只聚焦一章
4. 每章至少有一张便于扫读的表格
5. 洞察要写在前面，过程口吻不能进正式交付文档
6. 只要当前任务目录里已有可用真实产品图，本章就要优先下沉这些图，而不是只留在附录

## 5. 章节流水线

从现在开始，单章写作不再是“讨论一下就落文”，而是固定走 4 步：

1. `章节合同 agent`
   - 回填 `WRITING_PLAN.md`
   - 确定本章判断、证据、表格、配图、角标、阻塞项
2. `章节撰写 agent`
   - 只负责正文、表格、证据角标和图位需求
   - 不临时改章节职责
3. `配图 agent`
   - 优先选真实截图 / 抽帧 / 页面图
   - 再决定是否补 SVG / PNG 结构图
   - 负责图号、中文图注、插入位置和版式
4. `章节事实校验 agent`
   - 检查强判断、证据角标、图文对应、本地素材路径、中文化和留白问题

整篇通过后，再交给 `总编合稿 agent` 统一：

- 口径
- 图号
- 表格风格
- 重复段落
- `1. 总结`
- `10. 证据附录与索引`

## 6. 原则

- 先建模，再写章节
- 先章节讨论，再落文
- 先做章节合同，再开始正文
- 先做正文，再配图，再事实校验
- 总结最后写
- 证据附录单独补，不打断正文叙述
- 不要把公开证据直接上升成“内部战略意图”
- 高确定性标题必须匹配高置信证据
- 不要把“写作过程”“采集过程”“AI 怎么想的”写进正式正文

## 7. 章节最低配置

### 分析建模

- 分析模块表
- 建模图
- 每个模块的判断目标
- 证据入口角标

### 产品分析

- 产品架构图
- 核心功能矩阵
- 重点功能路径
- 关键功能详情分析
- 若已有真实界面截图，必须至少补一组 `对比板` 或 `路径板`
- 图注必须直接说明“这张图证明什么”

### 用户分析

- 样本说明
- 用户画像表
- 用户旅程图
- 用户需求 / JTBD 表
- 若真实界面可以对应用户角色，必须补一组 `角色板`

### 社媒分析

- 平台角色分工表
- 正向主题矩阵
- 负向主题矩阵
- 关键原话 / 关键链接

### SEO 与内容分析

- 关键词承接表
- 内容结构或承接格局图
- 若已有搜索页 / 落地页截图，优先下沉到本章

### 商业化分析

- 定价结构总表
- 权益边界说明
- 付费 / 认证 / 解锁节点分析
- 若已有价格页、会员页、认证页、权益页或打印/解锁节点截图，必须补一组 `节点板`

### 竞争判断

- 强项与短板表
- 护城河与脆弱点
- 跟 / 避 / 绕 / 观察
- 推荐有判断矩阵图

### 对我方建议

- 行动建议表
- P0 / P1 / P2
- 每条建议都要回指前文证据或章节

## 8. 配图要求

每章决定是否配图时，默认按这个优先级：

1. 真实截图 / 录屏抽帧 / 搜索页截图
2. 基于真实素材拼成的对比板 / 路径板 / 角色板 / 节点板
3. SVG / PNG 结构图

禁止：

- 用结构图替代本来就有的真实产品图
- 一张图承载多个互不相关的判断
- 图注只写“某某页面”而不写证明点
- 图放在附录，但正文不分析

## 9. 写作口径护栏

遇到以下写法时，默认先降级一档：

- `不是 X，而是 Y`
- `说明它在做……`
- `它的主战场不是……`
- `它当前更像……`

只有在证据足够时，才允许保留高确定性表述。否则应改写成：

- `从当前公开证据看，更接近……`
- `现阶段更像……，但仍需补充……验证`
- `用户可见层面呈现出……，不等于内部策略已经坐实`

## 10. 每章落文前再检查一次

- 这一段是事实，还是推断，还是建议
- 这句标题会不会比证据更绝对
- 这条判断有没有明确来源或章节回指
- 如果用户追问“为什么这么说”，是否能立刻指到证据
- 这章有没有出现“这份报告怎么写”“这轮采集怎么做”的过程口吻
- 这章的图是否已经下沉到正文，而不是停留在素材目录
- 图注是否中文且能直接说明判断
- 正文里是否有可点击角标或明确章节回指
