# 证据映射与降级策略

这份指南定义新版单产品竞品分析文档各章节的：

- 推荐证据来源
- 必填字段
- 最低可接受证据
- 证据不足时的降级写法

## 证据等级

- `A 级`：官网、产品页、定价页、帮助中心、官方公告、实测截图、录屏
- `B 级`：应用商店评论、公开用户评论、社媒帖子、论坛讨论、媒体采访、访谈摘要
- `C 级`：聚合文章、二手转载、非官方总结

默认规则：

- 关键事实优先用 `A 级`
- 用户观点至少要有 `B 级`
- `C 级` 只能辅助，不得单独支撑核心结论

## 1. 总结

证据来源：

- 依赖全篇汇总，不单独新增来源

必填字段：

- 一句话结论
- 为什么值得研究
- 对我方最重要的启发
- 优势与不足

最低证据：

- 至少来自后续 4 个不同章节的支撑

降级写法：

- 若仍有关键缺口，写“当前倾向判断”而不是“明确结论”

## 2. 分析建模

证据来源：

- 依赖全篇汇总，不单独新增来源

必填字段：

- 分析模块
- 为什么分析
- 希望得到的结论

最低证据：

- 每一行结论至少能回指 1 个正文章节

降级写法：

- 若某模块证据不足，写“当前无法确认 / 需补充页面或样本”

## 3. 产品分析

证据来源：

- 官网首页
- 产品功能页
- 帮助中心 / FAQ
- 实测截图 / 录屏
- 登录后页面截图

必填字段：

- 产品定位
- 目标用户
- 核心场景
- 一级模块
- 核心路径
- 转化节点

最低证据：

- 官网首页 + 至少一个功能页 + 至少一张有效截图

降级写法：

- 无法登录实测时，写“基于公开页面推断的结构判断”

## 4. 用户分析

证据来源：

- 评论数据
- 社媒帖子与评论
- 用户访谈 / 客服记录 / 群聊节选
- 官网案例与用户故事

必填字段：

- 至少 3 个细分群
- 每个细分群的行为模式
- 每个细分群的 JTBD
- 2-3 个 persona

最低证据：

- 至少 20 条有效用户反馈，且能覆盖 3 类差异化需求

降级写法：

- 样本不足时，写“初步分层”并标注样本量与偏差风险
- 不要轻易写成“它服务的不是 X，而是 Y”；更稳妥的是写“当前公开样本更偏向 X 类场景”

## 5. 社媒分析

证据来源：

- 小红书帖子与评论
- 应用商店评论
- 知乎 / B 站 / 微博 / Reddit / 论坛
- 媒体报道中的用户引用

必填字段：

- 正向主题 Top 列表
- 负向主题 Top 列表
- 高频问题与高严重度问题区分
- 代表性原话

最低证据：

- 至少 30 条有效评论或反馈样本

降级写法：

- 若平台覆盖不足，必须说明“结论可能偏向高活跃公开发声用户”

## 6. SEO 与内容分析

证据来源：

- 博客列表页
- 落地页
- 帮助中心
- 品牌词 / 产品词 / 问题词搜索结果
- 站内内容与内链结构

必填字段：

- 内容结构
- 关键词意图
- 承接页
- 转化连接方式

最低证据：

- 至少看到 1 个内容列表页 + 2 类关键词承接页面

降级写法：

- 若无法拿到自然流量数据，写“基于页面结构与 SERP 承接的策略判断”
- 不要仅凭公开承接面就写成“SEO 不是它的主战场”；优先写“当前公开搜索承接较弱”

## 7. 商业化分析

证据来源：

- 定价页
- 会员页 / 权益页
- 支付前页面
- 试用路径截图
- 用户关于价格与付费的评论

必填字段：

- 套餐结构
- 价格
- 权益边界
- 付费触发点
- 转化时机

最低证据：

- 至少有定价信息 + 权益边界 + 一条付费路径证据

降级写法：

- 无法拿到支付链路时，写“商业化判断主要基于定价结构与权益边界推断”
- 没有内部证据时，不要直接写“战略性免费产品”；优先写“用户可见层面暂未看到明确收费触发”

## 8. 竞争判断

证据来源：

- 前面所有章节输出
- 必要时补充 3-5 个直接竞品的基础事实

必填字段：

- 强项
- 短板
- 护城河
- 脆弱点
- 跟 / 避 / 绕 / 观察

最低证据：

- 每条战略判断至少能回指 2 个前文证据点

降级写法：

- 若直接竞品有限，可写“增加相邻替代方案辅助比较”

## 9. 对我方建议

证据来源：

- 来自前面所有章节的综合判断

必填字段：

- 用户策略建议
- 产品策略建议
- 增长策略建议
- 商业化策略建议
- 待验证动作

最低证据：

- 每条建议都要能回指前文证据

降级写法：

- 若证据仍不够强，使用“建议优先验证”而非“建议直接投入”

## 10. 证据附录

证据来源：

- 全部原始来源

必填字段：

- 来源索引
- 截图与录屏说明
- 评论与原话样本
- 不确定性说明

最低证据：

- 所有核心结论都能回到源头链接、截图或样本编号

降级写法：

- 链接无法公开时，保留来源名称、抓取日期、关键摘录

## 缺失证据的统一处理

遇到缺口时，必须使用以下规则：

1. 先写清“缺什么”
2. 再写“为什么缺”
3. 最后写“这会影响哪条判断”

推荐表达：

- `当前无法确认该产品真实付费转化表现，原因是未拿到支付前页面和转化数据，因此商业判断主要基于定价结构与用户反馈推断。`
- `当前用户分层仍属初步结论，因为公开反馈样本主要来自高活跃用户，尚未覆盖沉默用户。`
- `当前 SEO 判断未覆盖深层工具页，因此对内容转化能力的结论可信度中等。`

## 标题强度控制

章节小标题如果包含以下词，必须自查是否证据过强：

- `不是……而是……`
- `已经`
- `本质上`
- `主战场`
- `战略性`

如果只有公开页面、评论和体验素材，没有内部访谈或官方明示，应优先改成较低强度表述。

## 反模式

- 不要用单篇测评文章推出“用户都这么想”
- 不要把营销文案当用户需求
- 不要把评论数量等同于问题严重度
- 不要在证据不足时补想象
