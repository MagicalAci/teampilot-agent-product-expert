## 1. 本轮任务概况

- **平台**：搜索引擎（百度/Google SERP 承接分析）

- **样本量**：16 组关键词 × SERP 首页 Top 5 = 80 个 SERP 位置，去重后 46 个独立页面记录在 data.csv 中

## 3. 关键事实信号

### 3.1 官方 SEO 极弱——doubao.com 无爱学专属页面

`site:doubao.com 爱学` 的返回结果仅包含：

- `doubao.com` 主站首页
- `doubao.com/download/desktop` 桌面端下载页
- `doubao.com/legal/instructions` 算法备案页（含"豆包爱学大模型"备案编号）
- `team.doubao.com` Seed 团队博客

**没有任何 `doubao.com/student`、`doubao.com/aixue`、`doubao.com/education` 等面向 C 端用户的产品落地页。** 这与 Web 平台 Agent 发现的"C 端 Web 入口已撤"一致。

### 3.2 hippolearning.cn 仍被索引但服务 B 端

旧域名 `hippolearning.cn/introduction` 仍可访问且被搜索引擎索引，内容是"河马爱学"B 端产品介绍（定制学生平板 + 校园方案 + AI 大模型教育系统），与 C 端"豆包爱学" App 定位完全不同。

搜索"豆包爱学 官网"时，AIWW 将 `hippolearning.cn` 标注为官网入口，形成误导——用户点击后看到的是 B 端硬件产品页，不是 C 端 App 介绍。

### 3.3 SERP 被五类第三方页面全面占据

### 3.4 品牌混淆在搜索链路中普遍存在

三个维度的混淆：

**① "豆包" vs "豆包爱学"**  
搜索"豆包爱学 拍照答疑"→ SERP Top 4 中有 3 条写的是"豆包 App"拍照功能教程，不区分爱学。搜索"豆包爱学 闪退"→ 排名第一是"豆包 app 闪退"的解决方案。搜索"豆包爱学 收费"→ 前两条写的是"豆包 AI"收费政策，与爱学无关。

**② "豆包老师" 独立概念尚未建立**  
"豆包老师"搜索结果全是媒体文章（2024年9月密集报道），标题为"字节的教育新答卷：豆包爱学做了豆包老师"。没有独立产品页。这个功能名在搜索中完全依赖新闻稿承接。

**③ "豆包爱学" vs "河马爱学"**  
AIWW 仍将 hippolearning.cn 标为官网。搜索"豆包爱学 官网"时用户可能被导向 B 端旧站，看到的是学生平板和校园方案，而非 C 端 App。

### 3.5 AI 工具目录是最稳定的 SERP 承接层

ai-bot.cn、aigc.cn、aiww.com、1920m.com 这四个 AI 工具导航站在几乎所有搜索词中都占据 SERP Top 5。它们的共同特征：

- 页面发布于 2024 年左右，已有 1 年以上的索引历史
- 标题含"字节推出""豆包大模型""AI教育应用"等高关联关键词
- 内容结构标准化（产品介绍 + 核心功能 + 使用方法 + 应用场景 + 同类推荐）
- 有内链网络支撑（站内其他 AI 工具页互相引用）

这意味着，在官方缺位的情况下，**AI 工具目录站已经成为"豆包爱学"品牌搜索的事实承接层**。用户搜索后大概率点击这些页面获取产品信息。

### 3.6 对比类搜索由程序化平台占据

"豆包爱学 vs X" 类查询的 SERP 呈现两个特征：

- **36氪企服点评**（36dianping.com）占据结构化对比位——该平台程序化生成"A 和 B 哪个好"页面，甚至出现了"豆包爱学和通义千问哪个好"这样跨品类的对比
- **手游/App 评测站**（ai7.com, hncj.com）占据长尾对比位——这些站点的内容质量一般，但 SEO 做得好
- **教育垂直媒体**（jiaoyujie365.com）提供实测横评，内容质量较高

### 3.7 付费/会员信息搜索缺乏权威来源

搜索"豆包爱学 会员""豆包爱学 收费"时，SERP 中没有任何来自官方的定价页或会员说明页。用户看到的是：

- 淘宝自动生成页（内容为"豆包 AI"会员，非爱学）
- AIGC 导航站（仅提到"有付费会员服务，进阶版解题思路需开通"，无具体价格）
- App Store 页面（标记免费）

这构成一个搜索黑洞：用户有明确的付费决策意图，但搜索结果中找不到可信的定价信息。

## 4. 平台洞察与启发

### 4.1 官方在搜索引擎上几乎"隐身"

豆包爱学的 SEO 策略可以概括为：**不做 SEO**。

- 无独立产品官网（doubao.com 无爱学子站，hippolearning.cn 服务 B 端）
- 无官方博客、知识库、FAQ、帮助中心
- 无搜索引擎广告投放痕迹（搜索"豆包爱学"无 SEM 广告）
- 唯一官方可控的 SERP 存在是 App Store listing

这与字节的一贯策略一致：**依赖 App 生态内分发（抖音 → 豆包 App → 爱学功能模块），而非搜索引擎获客**。

### 4.2 "搜索链路"在豆包爱学的获客中可能权重极低

从上游平台数据看：

- 微博几乎无原生讨论 → 搜索引擎几乎不会从微博抓到爱学相关内容
- 小红书品牌混淆严重 → 即使小红书内容被搜索引擎索引，也分散在"豆包/豆包爱学/豆包老师"之间
- App Store 评分体量大（26 万评分/4.8 分） → 主要获客渠道是应用商店而非搜索引擎

**候选判断**：豆包爱学的获客链路是 "抖音 / 豆包 App 内推荐 → App Store 下载"，搜索引擎不是核心渠道。这是一个战略选择而非疏忽。

### 4.3 对竞品（MagicalAci）的启发

豆包爱学在搜索引擎上的"弱存在"意味着：

**① 品牌词截流机会**  
"豆包爱学 怎么样""豆包爱学 好用吗""豆包爱学 评测"等高意图词当前由低质量第三方页面承接。如果 MagicalAci 做高质量对比评测内容（如"豆包爱学 vs MagicalAci 深度对比"），有机会占据这些位置。

**② 问题词空白**  
"豆包爱学 家长"返回空结果——家长视角的搜索需求完全无人承接。这是一个内容机会。

**③ 长尾词竞争极低**  
"豆包爱学 数学 初中""豆包爱学 英语 口语"等学科+学段长尾词，SERP 仍然被 AI 导航站和 App Store 占据，没有深度内容。

## 5. 风险、噪音与缺口

### 5.1 噪音与风险

- **淘宝自动生成页**（bk.taobao.com）在多个查询中排名靠前，内容为程序化生成，混淆"豆包 AI"与"豆包爱学"，信息准确度低
- **"豆包爱学"裸品牌词 WebSearch 返回空**，可能是搜索引擎 API 行为（如 Google 的 Knowledge Panel 未返回文本结果），不代表真实 SERP 无结果。需浏览器打开搜索引擎验证
- 部分页面标题使用 emoji 和夸张表述（如淘宝百科），可能是低质量 SEO 内容

### 5.2 还缺什么


| 缺口       | 原因                             | 影响                                          |
| -------- | ------------------------------ | ------------------------------------------- |
| 关键词搜索量数据 | SerpAPI / Google Trends 待填 Key | 无法量化各关键词的真实搜索需求规模                           |
| 百度精确排名   | WebSearch 返回混合结果，无法区分百度/Google | 国内用户主要用百度，需单独验证百度 SERP                      |
| 外链/域名权威度 | 需 Ahrefs/Moz 等付费工具             | 无法评估第三方承接页的 SEO 根基强弱                        |
| 百度收录量    | 需 `site:` 在百度执行                | WebSearch 的 `site:doubao.com` 结果可能来自 Google |
| 付费搜索广告数据 | 需百度推广后台或 SemRush               | 无法确认是否有 SEM 投放                              |


### 5.3 建议补抓方向

- 用浏览器访问 `baidu.com` 逐词搜索，截图记录百度侧 SERP 首屏（尤其关注百度百科、百度知道是否有豆包爱学词条）
- 检查百度百科"豆包爱学"词条是否存在（这是百度品牌搜索最重要的承接位）
- 检查 `doubao.com` 的 robots.txt 和 sitemap，确认是否主动屏蔽了爬虫
