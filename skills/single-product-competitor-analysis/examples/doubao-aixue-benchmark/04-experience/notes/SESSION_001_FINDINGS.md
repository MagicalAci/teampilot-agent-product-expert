## 3. 关键事实信号

### 3.1 独立 App 首屏把“拍题答疑 / 作业批改”顶到最前

从首屏和抽帧可确认：

- 顶部直接展示年级选择，当前为 `五年级`
- 首屏最大卡片是 `拍题答疑`
- 第二大卡片是 `作业批改`
- 下方继续铺开 `AI错题本`、`英语作文`、`口语练习`、`拍照翻译`、`口算练习`、`绘本创作` 等模块

这说明豆包爱学不是把“AI老师”先做成抽象概念，而是把用户最常见的学习任务直接前置成可点击入口。

### 3.2 “AI讲题”不是一句 marketing 文案，而是实打实的交互价值时刻

从抽帧和用户反馈结合可确认：

- `frame-00030.png` 显示系统在进入讲解前，会先进入 `深度思考中...` 的中间状态，并显式展示它准备如何拆题
- `frame-00120.png` 显示正式讲解时，界面会按 `第一步：梳理已知条件` 这种结构化方式展开
- 录屏链路中可见它不是直接甩答案，而是把题目、条件和推导分开组织
- 用户主观反馈进一步补强了这一点：`可以随时打断，不懂还能继续说`

当前可形成的体验判断是：

- 它的 strongest moment 不是“算对一道题”，而是“把题拆开讲，而且你能中途打断它”
- 这种可打断、可追问、可继续解释的感觉，比单纯正确率更像真实老师

### 3.3 独立 App 里“讲题”与“做题”之间的切换很顺

从截图和抽帧看：

- 首页 -> 拍题答疑 -> 即将进入讲解 -> AI老师讲解页，这条链路非常短
- 讲题页底部保留 `按住说话` / 语音输入能力，强化了中途追问
- 顶部有题目展示区，中间是分步讲解区，界面角色分工清晰

这与用户反馈的“随时打断、不懂也可以继续问”形成一致证据。

### 3.4 主 App 里的爱学入口存在，但发现性偏弱

从主 App 截图可确认：

- `IMG_7936.jpg` 中，`豆包爱学` 以底部快捷入口 / 技能入口形式出现
- 用户明确说明：它之所以出现在靠前位置，是因为“我用得多了才在最前面”
- 进入后是爱学内容广场式页面：题目卡片、内容卡片、输入框，见 `IMG_7937.PNG`
- 从 `IMG_7938.PNG`、`IMG_7939.PNG` 看，主 App 进入爱学后也能进入讲解流程，但入口本身不像豆包主导航里的一级稳定入口

当前判断：

- 主 App 更像 `行为学习后的个性化前置入口`
- 这条路径有流量价值，但不适合当作“所有新用户都能自然发现”的高置信主入口

### 3.5 当前素材里未看到明确商业化触发

本轮截图和关键帧里：

- 未看到明确的 `会员`、`订阅`、`解锁高级功能`、`价格` 页面
- `frame-00300.png` 中，个人页更强调 `金币商城`、`任务中心`、`学习记录`、`AI错题本`
- 这更像留存/激励/习惯养成结构，而不是直接付费转化结构

当前可写成：

- 本轮实测素材里，`未见明确商业化触发`
- 至少在当前用户链路中，系统更强调先把用户带入 `讲题/练习/错题/记录` 的循环

## 4. 对当前主结论的影响

### 被强化的判断

- `免费 + 作业辅导/讲题` 是真实吸引力，不只是应用商店文案
- 豆包爱学的差异化不只是“会讲”，而是“分步 + 可打断 + 可继续解释”
- 主 App 的确在给爱学导流，但这条导流更像“个性化召回”，不是绝对强曝光入口

### 仍需补的缺口

- 还缺明确的 `会员 / 学力提升计划 / 付费页` 截图
- 还缺更完整的 `历史记录 / 学习记录 / 错题本详情` 页面链路
- 还缺一次更长的追问链，证明它是否能稳定保持上下文，而不是前几轮表现好

## 5. 下一步建议

1. 继续补 `设置 -> 学力提升计划 / 会员` 这条链路，优先找有没有价格和权益页
2. 补 `学习记录` 和 `AI错题本` 详情页，确认是不是有效留存闭环
3. 再补一条“连续追问更久”的录屏，重点看第 4-6 轮是否开始复读、偷懒或乱讲
