# 公开入口基线

> 产品：`豆包爱学`
> 任务：`doubao-aixue-overall-20260310-r2`

## 当前已经确认的公开入口事实

### 1. Web 端公开入口

- `doubao.com/student`：已确认是 `404`，不是渲染问题
- `doubao.com/chat/`：公开可访问，但非登录状态下是通用豆包聊天页，没有明确的“爱学”入口
- `doubao.com/about`：仅展示通用豆包能力，没有教育产品线承接
- `hippolearning.cn`：仍可访问，但定位偏旧品牌 `河马爱学` 的 B 端官网，不是当前 C 端主入口

### 2. App 端公开信号

- App Store 显示：`免费`
- App Store 描述里能看到的核心模块包括：
  - `AI 老师`
  - `拍题答疑`
  - `作业批改`
  - `写作助手`
  - `知识问答`
  - `成长挚友`
- 当前公开证据最强的吸引力仍是：`免费 + 作业辅导/讲题 + 英语/口语相关能力`

### 3. 商业化公开信号

- 当前没有公开 Web 定价页
- 已知线索显示 App 内可能存在 `学力提升计划` 或类似会员入口
- 但具体价格、权益、触发方式，仍必须靠 App 内实测补齐

## 这份基线对体验阶段的意义

- `Web` 端不能再作为主体验场景，它更像反面证据：说明豆包爱学 C 端主战场已收缩到 App
- 真实使用链路应优先从 `独立 App` 或 `豆包主 App 内爱学入口` 开始
- 这轮体验最重要的不是“还能不能打开网页”，而是验证：
  - 入口如何承接
  - 首次讲题是否真有价值
  - 连续追问会不会崩
  - 历史/错题能不能形成闭环
  - 付费入口到底藏在哪

## 需要用户实测补齐的关键缺口

1. `独立 App` 首屏到底把用户带向哪个主任务
2. `首次拍题/讲题` 的真实流程和结果质量
3. `连续追问` 时是否会复读、丢上下文或开始乱讲
4. `英语/口语` 入口是否真的被重点强调
5. `历史/错题/结果` 是否能沉淀为留存闭环
6. `学力提升计划 / 会员` 的实际页面、价格、权益和触发文案
7. `豆包主 App 内爱学入口` 与独立 App 的导流关系
