# Cursor 交互模拟

这份文档演示三条指令在真实对话中的推荐节奏：

- `/竞品分析`
- `/爬取`
- `/竞品引导`

## 1. `/竞品分析` 全流程模拟

### 用户输入

```markdown
/竞品分析 Duolingo
分析目标：判断它为什么留存高，以及哪些做法值得 iXue 借鉴。
```

### AI 第一轮

```markdown
我先不直接开始搜，先把这次任务边界对齐。

我理解这次要研究的是 Duolingo，目标是搞清：
1. 它为什么留存高
2. 哪些做法值得 iXue 借鉴

我这边先直接按默认规则走：
1. 完整章节顺序推进
2. 默认纳入 App 深度体验
3. 默认按最高标准交付

这一轮我只需要你补两个真正会影响方向的信息：
1. 这轮你特别重点看什么
2. 有没有什么特殊限制、禁区，或者你明确想让我额外深挖的角度

如果后面要写“对我方建议”但上下文里还没有我方产品信息，我会在写作前再补问，不会卡在开头。
```

### 用户补充后

AI 应做两件事：

- 创建任务文件夹骨架
- 创建新的独立任务 ID 和独立任务目录
- 把刚才的结论补进 `PROCESS_LOG.md` 和 `KEY_DECISIONS.md`
- 后续主报告默认写到当前任务目录 `06-writing/<task-slug>.md`

### 进入关键词核对

```markdown
我先给这次搜索的关键词骨架，你补一轮：

1. 核心词
2. 别名词
3. 功能词
4. 长尾词
5. 评价词
6. 问题词
7. 竞品词
```

### 进入并发采集

```markdown
关键词我收到了。接下来我不会自己一条线乱搜，而是先把搜索子代理拆出来：

Batch 1：
- 官网结构 Agent
- App Store Agent
- 小红书 Agent
- 微博 Agent

Batch 2：
- B站 Agent
- 知乎 / 问答 Agent
- SEO 承接 Agent
- 商业化 Agent

每个平台都会沉淀：
- `03-platforms/<platform>/summary.md`
- `03-platforms/<platform>/data.csv`
```

### 进入审核门禁

```markdown
搜索子代理跑完后，我不会直接写主报告。

我会先交给审核代理检查：
1. 子代理有没有漏跑
2. 平台产物是否齐全
3. 样本量是否达标或已取得你确认
4. 有没有偷跑到正式报告

如果不合格，我会明确打回给具体子代理；最多循环 5 轮。
```

### 进入真实体验

```markdown
采集结果我先整理好了，你先帮我看：
1. 哪些平台结果你觉得可信
2. 哪些地方噪音大
3. 还要不要补抓

如果这一轮没问题，我接着带你做真实体验。
```

### 进入写作规划

```markdown
资料和体验都差不多齐了。

接下来我不直接出全文，先和你一起定写作顺序：
1. 分析建模
2. 产品分析
3. 用户分析
4. 社媒分析
5. SEO 与内容分析
6. 商业化分析
7. 竞争判断
8. 对我方建议
9. 总结
10. 证据附录
```

### 进入事实核查

```markdown
正文写完后，我不会自己手动核一遍就算结束。

我会按三轮分别调用独立只读 `research-review` 子代理：
1. Round 1 核事实和来源
2. Round 2 核判断和证据强度
3. Round 3 核附录索引和主报告路径

只有三轮都落盘完成，才会进入 SVG 补图。
```

## 2. `/爬取` 模拟

### 用户输入

```markdown
/爬取 小红书 Duolingo
目标：先抓学生和家长视角的真实讨论。
```

### AI 第一轮

```markdown
我先确认这次采集边界。

我会默认新建一个独立采集任务，不会直接接到旧任务。

请你确认三件事：
1. 这次只抓小红书，还是还要补 App Store / 微博 / B站
2. 样本量默认要不要按至少 200 条有效样本执行
3. 有没有明确要参考的旧任务或旧资料；如果没有，我就按全新任务处理
```

### AI 进入执行后

```markdown
这一轮我会做三件事：
1. 明确关键词和长尾词
2. 用现有 MCP / 搜索能力抓数据
3. 输出平台 `summary.md + data.csv`

如果样本量不够，我会回来先问你，不会直接硬写结论。
```

## 3. `/竞品引导` 模拟

### 用户输入

```markdown
/竞品引导 Duolingo
重点引导我体验核心学习路径和会员触发节点。
```

### AI 第一轮

```markdown
我先给你排体验顺序，不让你乱逛：

1. 首页/入口页
2. 首次完成一个核心学习任务
3. 登录后主界面
4. 会员触发节点
5. 历史记录或结果页

每一步我都会告诉你：
- 为什么先看这个
- 要录什么
- 要截什么
- 体验后你应该想什么
```

### AI 追问洞察

```markdown
这一段体验完后，你先别急着继续。

我想问你 4 个问题：
1. 你觉得哪里最顺
2. 你觉得哪里最卡
3. 有没有反直觉但有效的设计
4. 有没有你觉得“很想学”或“必须避开”的点
```

### 素材处理

AI 拿到录屏后，应继续说明：

```markdown
我会把这段录屏逐帧切图，存到任务文件夹里。
后面做产品分析、商业化分析和证据附录时都会复用这些素材。
```

## 4. 整体分工

| 类型 | AI负责 | 用户负责 |
|---|---|---|
| 重复劳动 | 建目录、组织采集、切帧、归档、校验 | 提供素材 |
| 分析过程 | 追问、归纳、补盲区、组织正文 | 给判断和洞察 |
| 最终决策 | 给候选建议和风险提醒 | 最终拍板 |
