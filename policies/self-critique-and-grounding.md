# 自检与接地协议（Self-Critique & Grounding）

这是产品专家 Agent 的**统一质量闸**——五项能力在"**出结论 / 成稿 / 交付前**"都过这一关。它合并了原本散落在多处、本质同一件事的四块：**自检修订**（推理范式角度）、**幻觉缓解/接地引用**（安全角度）、**reviewer 高层验证**（编排角度）、**完成判定**（评估角度）。统一成一份，避免各能力各写一套、互相打架。

> 合并来源：`findings/01`（Self-Refine/CoVe/Reflexion）+ `findings/10`（grounding/abstain/不确定性）+ `findings/02`（通用 reviewer 契约、防过早终止）+ `findings/09`（完成判定）。research 的三轮事实核查是本协议的"满血实现样板"。

---

## 0. 何时触发（不要每步都自检）

**reflect-when-stuck**：不是每一步都反思——那样又慢又贵。仅在以下时机过闸：
- 产出**面向用户的结论/交付物**前（PRD 章节、竞品判断、SQL 口径与数字、Demo 交付、调研成稿）；
- 自评**不达标**或**受阻**时进入修订；
- **高风险/不可逆**结论（对外资产、关键判断）必过。

低风险的中间步、确定性子任务**不过闸**，保持顺畅。

---

## 1. 三步闸：自检 → 接地 → 完成判定

### ① 自检修订（Self-Refine，最大 2–3 轮）
按该任务的**质量维度**自评（如：是否答全问题、口径是否一致、是否有未支撑的强断言、格式是否合规），发现问题就改；**最大重试 2–3 轮**（与门禁"最多 5 轮"对齐），不达标则降级表达或转人工，**不空转**。

### ② 接地（grounding，事实型结论必做）
- **强制引用**：事实型结论默认回指证据（证据文件 / 采集来源 / 检索命中），无证据**不下强结论**。
- **CoVe（Chain-of-Verification）**：对事实密集输出，先答 → 自生成核查问题（"这条数据来源是？口径一致吗？是否过期？"）→ 据此核对修订。
- **检索接地回路**：对每条强判断，从证据池召回候选 → LLM 按忠实度重排 → 引用最被支撑者（走 `retrieval-protocol.md`）。
- **abstain（弃权）/ 不确定性表达**：没把握就**显式标"待核实"或弃权**，而非编造；高风险结论给**置信度 + 前提**。对齐仓库既有"信息不足澄清而非编造"。

### ③ 完成判定（防过早终止 + 高层验证）
独立 reviewer 视角，按 **checklist 对照 `brief.md` 目标逐项验证**——是**高层目标级验证**（"真达成目标了吗"），不是表层"产物存在吗"（防 MAST FM-3.3 表层验证 / FM-3.1 过早终止）：
- [ ] 任务目标逐项达成（对照 brief）
- [ ] 关键结论都有接地引用 / 不确定项已标注
- [ ] 硬约束/口径未违反
- [ ] 格式与交付结构合规
- [ ] 未决项、风险、剩余待确认已分层表达
**任一项不达标 → 不得终止**，回到 ① 或转人工。

## 2. 各能力接入点（统一引用本协议）

| 能力 | 过闸时机 | 重点 |
|---|---|---|
| 调研分析 | 成稿前 + 三轮事实核查 | 接地引用 + CoVe（已有满血实现） |
| 产品策划 | PRD 关键章节定稿前 | 完成判定（对照需求）+ 强断言接地 |
| AI 策划 | 评测/调优结论前 | 不确定性表达 + 完成判定 |
| Demo 开发 | 交付/校验前 | 完成判定（功能/交接齐备）|
| SQL 查询 | 输出口径与数字前 | **强制接地（SQL 与数字一致）+ Self-Consistency**（高风险口径多跑取一致）|
