# 提交与评审契约

## 提交时必须包含

- 最终交付物
- 对话摘要或关键过程记录
- 决策与理由
- 洞察与后续建议
- 对应能力的标准目录结构与关键脚本校验结果

## 能力特定要求

### 单产品分析

- 必须保留任务卡、平台 summary/data、体验报告、章节成稿与事实核查记录
- 关键判断必须能回指到证据文件或事实核查轮次

### 产品策划

- 必须保留 PRD、图文资产、证据包与 review 交接信息
- 如果执行的是 `/产品策划校验`，需要明确列出缺口、风险和修正建议

### AI策划

- 必须保留 AI PRD、脚本或样例、测试报告
- 如果有开发交接诉求，还需补 README 或脚本包清单

## 评审时重点确认

- 是否真正回答了任务目标
- 是否保留了足够证据与判断依据
- 是否存在应回写到 Agent 仓库的方法论增量
- 是否通过对应能力的 smoke test、校验脚本或结构完整性检查
