# 成本纪律方法论（Cost Discipline）

这是产品专家 Agent 的**成本纪律总纲**——把 `agent-team-methodology.md` 第四部分"Token 优化"的**原则**升级为**可执行落地约定**。它不改变"做什么"，只优化"用什么档位、是否复用算过的结果、花多少 token/调用/时间把活办成"。本仓库是 **Cursor 原生、无自有 serving**，所以 P0 全是**文件化纪律**，重 serving（vLLM 等）一律 P2。

> 细节编目见 `research/ai-capability-upgrade/findings/12-model-routing-cost.md`（70 条 SOTA）。**本次研究三次 `resource_exhausted` 事故，正是"无成本/预算护栏"的反面教材，已固化为 §3。**

---

## 1. 模型分级决策表（任务 → 档位 + 是否开思考）

**默认中低档、不开最高思考；只有规划/裁决/核查才上高档。**

| 子任务 | 档位 | 思考 | 谁来做 |
|---|---|---|---|
| 归一/分类/关键词扩展/平台 summary/配图提示 | 低 | 关 | 子代理低档 |
| PRD 章节撰写 / 跨源综合 / 一般生成 | 中 | 按需 | 主代理中档 |
| 任务规划 / 三轮事实核查裁决 / 竞品定性 / SQL 口径 | 高 | 开 | 主代理高档 |

- **思考预算档位**：推理模型的"思考 token"按输出价计费却不可见，最高档比最低档可贵 30–80×——**默认不要用最高思考档**，按任务难度调。
- 对标 GPT-5 router / RouteLLM / FrugalGPT 的成本-质量思想。

## 2. 级联式派发（先弱后强、强验弱）

把"扇出/扇入"细化为成本版：**采集/归一/分类用低档子代理产出，主代理用高档仅做裁决/校验**（生成-检验式，对标 FrugalGPT/UCCI）。别让判断和裁决都用同一高档跑全程。

## 3. 任务级预算护栏（最高优先，直接呼应限流事故）

给多代理/扇出任务设**硬上限**：
- 单批并发子代理 **≤ 4**；单任务累计子代理数上限（如 ≤ 8，超过需理由）；
- 最大联网调用数 / 最大重试数 / 最大编排轮次；
- **超限收敛**：到上限即"**停止新派、用当前已得最佳结果交付 + 标注未尽项**"，而非空转或反复重试。

**限流即停用不重试（铁律）**：外部工具/MCP 报限流（`resource_exhausted`）或鉴权失败（如 tavily `Invalid API key`）→ **立即停用该工具、降级备用通道或纯知识，禁止反复重试**。反复重试失效工具会加剧限流、放大成本事故（本次三次 `resource_exhausted` 的直接成因）。

`task-navigator.mdc` 高扇出阶段规划时附"预估调用量/档位/是否可复用旧产物"。

## 4. 缓存复用门禁（三层）

- **前缀稳定化**：稳定内容（命令路由/能力库/案例库/系统提示）前置、任务变量后置，吃满 IDE/模型侧 **prompt 前缀缓存**（命中省 50–90%）。
- **结果缓存**：昂贵同参调用按"工具名+入参+日期窗"缓存（见 `tool-use-protocol.md` §5）。
- **读旧产物门禁**：开工前先查近期同对象产物（如 `tasks/*/.../summary.md`）——**新鲜则增量补采，过期/不存在才全量**；把"全量重爬"从默认改为显式选择（改 `research-toolkit/protocols/subagent-search.md`）。

## 5. Batch / 错峰意识
可等的离线活走 Batch API（约 5 折）；非实时任务不必抢同步配额。
