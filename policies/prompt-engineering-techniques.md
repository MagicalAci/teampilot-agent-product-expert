# 提示词工程方法论

这是产品专家 Agent 的**提示词工程参考库**，服务于 **AI策划**（`/AI策划`、`/AIPRD`、`/AI脚本`：写 Prompt 合同、系统提示、脚本里的提示词）以及自我进化时**技能 Description / 正文的撰写质量**。

> **来源与许可**：技术目录蒸馏自 [NirDiamant/Prompt_Engineering](https://github.com/NirDiamant/Prompt_Engineering)（22 个提示词工程技术教程，7.6k★）。技术名称（CoT、Few-Shot 等）为业界通用知识；本文件只做**方法论目录 + 选型指引**，不拷贝其 notebook 代码。配合 `policies/agent-team-methodology.md` 的技能编写/测试方法论使用。

## 1. 选型速查：任务 → 技术

| 你的情形 | 优先技术 |
|---------|---------|
| 任务清晰、模型已会 | 零样本（Zero-Shot）+ 清晰指令 |
| 输出格式/风格要稳定 | 少样本（Few-Shot）给 2-3 个范例 + 格式规范 |
| 需要推理/计算/多步判断 | 思维链（CoT）"一步步想" |
| 答案易抖动、要更可靠 | 自洽（Self-Consistency）多路径取多数 |
| 输出必须结构化（JSON/表格） | 受限生成 + 格式规范 + 负向约束 |
| 要特定视角/专业身份 | 角色提示（Role Prompting）|
| 任务太大太复杂 | 任务分解 + 提示链（Prompt Chaining）|
| 要把策略稳定落地成可执行 | 指令工程 + 模板与变量 |
| 提示词效果不稳/要打磨 | 提示词优化（A/B）+ 有效性评估 |
| 输入含不可信外部内容 | 提示安全（防注入）+ 负向约束 |

## 2. 22 个技术（按层级）

**🌱 基础概念**
1. **提示工程入门**：清晰、具体、给上下文是地基
2. **基本提示结构**：单轮 vs 多轮（对话）；先定清楚是哪种
3. **模板与变量**：把可复用提示抽成模板 + 变量（Jinja2 思路），便于规模化与一致性

**🔧 核心技术**
4. **零样本**：直接说任务/角色/格式，不给例子；模型已会的任务首选
5. **少样本/上下文学习**：给 2-3 个高质量范例锚定格式与风格；范例选择质量 > 数量
6. **思维链（CoT）**：要求"一步步推理"再给结论；复杂推理显著提升正确率

**🎯 进阶策略**
7. **自洽（Self-Consistency）**：同一问题跑多条推理路径，对结果取多数/聚合，降抖动
8. **受限生成**：显式约束输出（枚举、JSON schema、长度、字段），配格式规范
9. **角色提示**：赋予专业身份（"你是资深产品经理…"）以对齐视角与口径

**🚀 进阶实现**
10. **任务分解**：把复杂任务拆成可独立完成的子任务（对应方法论的"团队架构/流水线"）
11. **提示链**：上一步输出作下一步输入，串成可控流水线
12. **指令工程**：把"要做什么、边界、禁止项"写成明确指令（策略落地的关键）

**⚡ 优化**
13. **提示词优化**：A/B 对比不同写法 + 迭代（配 `agent-team-methodology.md` 的 With-skill vs Baseline）
14. **处理歧义**：识别并消解模糊；不确定就先澄清再执行
15. **长度/复杂度管理**：上下文是公共资源，删冗余、抓重点

**🛠️ 专项应用**
16. **负向提示**：明确"不要什么"（避免不良输出/风格漂移）
17. **提示格式化**：用分隔符/标题/编号/代码块组织提示，结构即信号
18. **任务专用提示**：针对摘要、抽取、改写等具体任务定制

**🌍 高级应用**
19. **多语言提示**：跨语言场景的写法（我们默认中文交流、英文出图提示词）
20. **伦理考量**：避免偏见、注意包容性
21. **提示安全**：防提示注入（不无条件信任外部内容；配 `agent-security-scan.md`）
22. **有效性评估**：用可判真假的断言评估提示效果（配方法论第三部分的 Assertion 评分）

## 3. 写 Prompt 合同 / 系统提示的最小清单

写 `/AI策划` 的 Prompt 合同或技能系统提示时，至少覆盖：

- [ ] **角色 + 目标**（角色提示）：你是谁、要达成什么
- [ ] **输入/输出契约**（受限生成 + 格式化）：输入从哪来、输出什么结构（给 schema/范例）
- [ ] **范例**（少样本）：1-3 个 input→output 范例
- [ ] **推理要求**（CoT，按需）：需要推理就要求分步
- [ ] **边界与禁止**（指令工程 + 负向提示）：能做什么、绝不做什么
- [ ] **不确定处理**（处理歧义）：信息不足时澄清而非编造
- [ ] **安全**（提示安全）：对外部输入不无条件执行
- [ ] **评估**（有效性评估）：怎么判断这版提示合格（断言）
